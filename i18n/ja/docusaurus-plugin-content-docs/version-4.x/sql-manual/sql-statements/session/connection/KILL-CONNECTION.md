## デスクリプション

指定された接続IDを持つ接続を終了します。これにより、この接続に関連付けられたクエリも終了されます。

## Syntax

```sql
KILL [ CONNECTION ] <connection_id>
```
## 必須パラメータ

**<connection_id>**

> 接続のIDです。SHOW PROCESSLIST文を通じてクエリできます。

## アクセス制御要件

このSQLコマンドを実行するユーザーは、接続を所有するユーザーであるか、少なくとも`ADMIN_PRIV`権限を持っている必要があります。

## 例

`connection_id`をクエリする：

```sql
show processlist;
```
結果：

```sql
+------------------+----+------+-----------------+---------------------+----------+------+---------+------+-------+-----------------------------------+------------------+--------------+--------------+
| CurrentConnected | Id | User | Host            | LoginTime           | カタログ  | Db   | Command | Time | State | QueryId                           | Info             | FE           | CloudCluster |
+------------------+----+------+-----------------+---------------------+----------+------+---------+------+-------+-----------------------------------+------------------+--------------+--------------+
| Yes              | 16 | root | 127.0.0.1:63746 | 2024-11-04 20:18:07 | internal | test | Query   | 0    | OK    | e4d69a1cce81468d-91c9ae32b17540e9 | show processlist | 172.16.123.1 | NULL         |
+------------------+----+------+-----------------+---------------------+----------+------+---------+------+-------+-----------------------------------+------------------+--------------+--------------+
```
KILL コマンドを送信:

```sql
KILL CONNECTION 16;
```
