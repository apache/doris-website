## 説明

このステートメントは、テーブル内のインデックスに関する情報を表示するために使用されます。現在、inverted indexesとann indexesのみがサポートされています。

## 構文

```SQL
SHOW INDEX [ ES ] FROM [ <db_name>. ] <table_name> [ FROM <db_name> ];
```
## Varaint構文

```SQL
SHOW KEY[ S ] FROM [ <db_name>. ] <table_name> [ FROM <db_name> ];
```
## 必須パラメータ

**1. `<table_name>`**: インデックスが属するテーブル名。

## オプションパラメータ

**1. `<db_name>`**: データベース名、オプション。指定されない場合、デフォルトで現在のデータベースが使用されます。

## 戻り値

| カラム名       | 型     | 説明                                                                                                           |
|----------------|--------|----------------------------------------------------------------------------------------------------------------|
| Table          | string | インデックスが配置されているテーブル名。                                                                        |
| Non_unique     | int    | インデックスが一意かどうかを示します: <br> - `0`: 一意インデックス <br> - `1`: 非一意インデックス                 |
| Key_name       | string | インデックス名。                                                                                                |
| Seq_in_index   | int    | インデックス内でのカラムの位置。このカラムはインデックス内でのカラムの順序を示し、複数のカラムが複合インデックスを形成する際に使用されます。 |
| Column_name    | string | インデックス化されたカラム名。                                                                                   |
| Collation      | string | インデックスカラムのソート順序: <br> - `A`: 昇順 <br> - `D`: 降順。                                             |
| Cardinality    | int    | インデックス内の一意な値の数。この値はクエリ効率の推定に使用され、値が高いほどインデックスの選択性とクエリ効率が高くなります。 |
| Sub_part       | int    | インデックスが使用するプレフィックス長。インデックスカラムが文字列型の場合、`Sub_part`はインデックスの最初の数文字の長さを表します。 |
| Packed         | string | インデックスが圧縮されているかどうか。                                                                           |
| Null           | string | `NULL`値が許可されているかどうか: <br> - `YES`: NULL値許可 <br> - `NO`: NULL値不許可                            |
| Index_type     | string | インデックスの種類: <br> - `BTREE`: B+ツリーインデックス（MySQLのデフォルト型） <br> - `HASH`: ハッシュインデックス <br> - `RTREE`: R-ツリーインデックス <br> - `INVERTED`: 転置インデックス（全文インデックスなど） |
| Comment        | string | インデックスのコメントまたは説明、通常はカスタム備考。                                                           |
| Properties     | string | インデックスの追加プロパティ。                                                                                   |


## アクセス制御要件

このSQLコマンドを実行するユーザーは、少なくとも以下の権限を持つ必要があります：

| 権限       | オブジェクト | 備考 |
|:-----------|:-------------|:-----|
| SHOW_PRIV  | Database     |      |

## 例

- 指定した`table_name`のインデックスを表示

     ```SQL
      SHOW INDEX FROM example_db.table_name;
     ```
