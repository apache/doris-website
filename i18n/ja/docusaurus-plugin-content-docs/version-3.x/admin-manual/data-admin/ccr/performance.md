このドキュメントのパフォーマンスデータはデフォルト設定に基づいています。高いネットワークレイテンシや高スループットの書き込みシナリオに直面している場合は、最適化について[Operation Manual](manual.md)を参照してください。

## テストデータセット
- **データセット**: TPC-H 1T

## テストクラスタ設定

| 設定項目 | アップストリーム設定                     | ダウンストリーム設定                     |
|--------------------|-------------------------------------------|---------------------------------------------|
| FE                 | 2 cores 16 GB                             | 2 cores 16 GB                               |
| BE                 | 3 nodes, 16 cores 64 GB, 3*500 GB per node | 3 nodes, 16 cores 64 GB, 3*500 GB per node |

---

## インクリメンタル同期パフォーマンステスト

### テスト手順

1. アップストリームクラスタでTPC-H 1T用のデータベースとテーブルを作成します。
2. TPC-H 1Tデータベースの同期ジョブを作成します。
3. TPC-H 1Tデータインポートの完了を待ち、完了時間を記録します。
4. ダウンストリームデータ同期の完了を待ち、完了時間を記録します。

### テスト結論
インクリメンタル同期時間差：33秒

---

## フル同期パフォーマンステスト

### テスト手順
1. アップストリームクラスタでTPC-H 1Tのライブラリとテーブル情報を作成し、データインポートを完了して、完了時間を記録します。
2. TPC-H 1Tデータベースの同期ジョブを作成します。
3. ダウンストリームデータ同期の完了を待ち、完了時間を記録します。

### テスト結論
フル同期時間差：6分1秒

---

## Flink同期パフォーマンステスト

### テスト手順
1. Flinkを使用してアップストリームに100,000,000レコードをインポートします。
2. ライブラリとテーブルの同期ジョブを作成します。
3. 各段階（例：1,000,000レコード、2,000,000レコードなど）でダウンストリーム同期完了時間とアップストリームインポート完了時間の差を観察します。
4. アップストリームでの最後のインポート完了時間を記録します。
5. ダウンストリーム同期完了時間を記録します。

### テスト結論
各段階での**遅延時間**は`5秒`以内に維持されます。
