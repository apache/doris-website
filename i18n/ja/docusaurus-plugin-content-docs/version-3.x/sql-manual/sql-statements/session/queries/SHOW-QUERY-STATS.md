## 説明

このステートメントは、履歴クエリによってヒットしたデータベースtableカラムを表示するために使用されます

## 構文

```sql
SHOW QUERY STATS [ { [FOR <db_name>] | [FROM <table_name>] } ] [ALL] [VERBOSE]];
```
## オプションパラメータ

**1. `<db_name>`**

> このパラメータが設定されている場合、データベースのマッチング情報が表示されます

**2. `<table_name>`**

> このパラメータが設定されている場合、Tableのマッチングステータスが照会されます

**3. `ALL`**

> ALLはすべてのインデックスのマッチング情報を表示するかどうかを指定します

**4. `VERBOSE`**

> VERBOSEは詳細なマッチング情報を表示します

## アクセス制御要件

このSQLコマンドを実行するユーザーは、少なくとも以下の権限を持っている必要があります：

| 権限               | オブジェクト | 備考                                    |
|:-------------------|:-------------|:---------------------------------------|
| SELECT_PRIV        | DATABASE     | 照会するデータベースに対するSELECT権限を持っている必要があります |

## 使用上の注意

- データベースとTableの履歴クエリマッチングステータスを照会します。feの再起動後にデータはリセットされ、統計情報は各feごとに収集されます。

- FOR DATABASEとFROM TABLEを使用して、照会するデータベースまたはTableのマッチング情報を指定できます。その後にデータベース名またはTable名を続けます。

- ALLおよびVERBOSEパラメータは単独で使用することも組み合わせて使用することもできますが、必ず最後に使用し、Tableクエリに対してのみ使用する必要があります。

- 使用中のデータベースがない場合、単に'SHOW QUERY STATS'を実行するとすべてのデータベースのヒット数が表示されます。

- ヒット結果には2つの列がある場合があります：QueryCountは列が照会された回数を示し、FilterCountは列がwhere条件として照会された回数を示します。
