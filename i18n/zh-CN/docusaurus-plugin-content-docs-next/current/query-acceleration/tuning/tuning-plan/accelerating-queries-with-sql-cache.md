## 概述

SQL Cache 是 Doris 提供的一种**查询结果缓存机制**，通过缓存 SQL 查询的结果集来加速重复查询，显著提升查询性能。

### 适用性检查清单

在使用 SQL Cache 前，请先确认以下条件：

-   [ ] 查询语句**重复执行**频率较高
-   [ ] 数据**更新频率较低**（缓存命中率高）
-   [ ] 查询**不包含**随机函数（如 `random()`）
-   [ ] 每次查询的**指标字段一致**（不会动态增减）

> 关于 SQL Cache 详细实现原理，请参考 [查询缓存（SQL Cache）](../../../query-acceleration/sql-cache-manual) 章节。

## 使用限制与对比

<!-- 知识类型：限制说明 -->
<!-- 适用场景：选择合适的缓存策略 -->

下表列出了 SQL Cache 的关键限制条件：

| 限制项           | 说明                                                                                       | 影响 / 错误表现                       |
| ---------------- | ------------------------------------------------------------------------------------------ | ------------------------------------- |
| 随机函数查询     | 查询中包含 `random()` 等随机值生成函数时，不应使用 SQL Cache                               | 缓存命中后返回相同结果，失去随机性     |
| 指标列扩展查询   | 已缓存 2 个指标的结果，**不能**用于满足查询 3 个指标的请求                                 | 缓存无法复用，需重新执行查询           |
| 数据高频更新场景 | 数据频繁变更时，缓存频繁失效                                                               | 缓存命中率低，加速效果不明显           |

## 最佳实践

<!-- 知识类型：操作建议 -->
<!-- 适用场景：调优 SQL Cache 以获得最佳性能 -->

-   **目的**：提升查询性能 → **方式**：在数据更新频率较低的场景中启用 SQL Cache → **说明**：缓存命中率越高，加速效果越显著。
-   **目的**：避免错误结果 → **方式**：避开包含随机函数或动态指标的查询 → **说明**：上述场景缓存可能返回非预期结果。
-   **目的**：获得最佳收益 → **方式**：根据数据特征和查询模式调整缓存参数 → **说明**：参数调整方法详见 [查询缓存（SQL Cache）](../../../query-acceleration/sql-cache-manual)。

:::tip 提示
SQL Cache 不适用于包含 `random()` 等随机函数的查询，也不支持用「少指标缓存」满足「多指标查询」。在数据更新频率较低的场景中，合理配置可显著提升查询性能。
:::

## FAQ

<!-- 知识类型：常见问题 -->
<!-- 适用场景：使用过程中的疑问排查 -->

**Q1：SQL Cache 适用于哪些场景？**
A：适用于**查询语句重复**且**数据更新频率较低**的场景，缓存命中率越高加速效果越好。

**Q2：为什么含 `random()` 的查询不能使用 SQL Cache？**
A：`random()` 每次执行应返回不同结果，使用缓存会让结果保持不变，失去随机性。

**Q3：之前缓存了 2 个指标，现在查询 3 个指标能复用缓存吗？**
A：不能。当前不支持用部分指标的缓存结果满足更多指标的查询请求，需重新执行查询。

**Q4：如何调整 SQL Cache 参数以获得最佳性能？**
A：需结合具体的数据特征和查询模式调整，详见 [查询缓存（SQL Cache）](../../../query-acceleration/sql-cache-manual) 章节。

## 总结

SQL Cache 是 Doris 提供的查询优化机制，能显著提升重复查询的性能。在**数据更新频率较低**的场景中合理使用，并避开**随机函数**与**动态指标**等限制场景，即可获得最佳的加速效果。
