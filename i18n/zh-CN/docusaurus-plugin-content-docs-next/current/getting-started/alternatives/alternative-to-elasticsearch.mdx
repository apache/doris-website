Elasticsearch 与 Apache Doris 在可观测性、网络安全和实时分析领域均有广泛应用。Elasticsearch 在存储资源与写入成本方面存在较高开销，相比之下，Apache Doris 利用其高效存储架构、存储压缩比等特性显著降低了资源消耗，并提供了更完善的分析能力以及更优的查询性能。


## 何时选择 Apache Doris

| 场景 | 推荐选择 | 原因 |
|------|----------|------|
| 需要更低的存储成本 | **Apache Doris** | 压缩比 1:5-1:10，远优于 Elasticsearch 的 1:1.5 |
| 需要复杂分析能力 | **Apache Doris** | 支持多表 Join、物化视图、标准 SQL，无需学习专用 DSL |
| 需要更低的总体拥有成本 | **Apache Doris** | 支持存算分离架构，弹性扩展更灵活 |
| 需要更高写入吞吐量 | **Apache Doris** | 写入性能损耗低于 10%，Elasticsearch 可高达 3 倍 |
| 主要做日志检索和简单聚合 | 两者皆可 | 根据团队熟悉度选择 |
| 已有 Elasticsearch 稳定运行 | 继续用 Elasticsearch | 迁移有成本，收益需评估 |

## 常见问题

**Q: 如何从 Elasticsearch 迁移到 Apache Doris？**

A: Apache Doris 提供多种数据导入方案，支持从 Elasticsearch 全量迁移数据。具体步骤请参考[数据导入文档](../data-baseline/load-manual)。

**Q: Doris 的学习曲线如何？**

A: Doris 支持标准 SQL，使用 MySQL 协议连接，对于熟悉 SQL 的工程师无需额外学习查询语言。

**Q: Doris 和 Elasticsearch 的日志检索性能差异？**

A: 两者在简单关键词检索场景性能相近，但 Apache Doris 在聚合分析、时序数据分析场景下性能更优，且支持更丰富的分析函数。
