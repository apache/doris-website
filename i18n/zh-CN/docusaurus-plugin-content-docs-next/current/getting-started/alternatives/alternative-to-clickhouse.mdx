Apache Doris 和 ClickHouse 都是全球领先的实时分析型数据库，均支持列式存储与极速查询。Apache Doris 在三个关键领域具有显著优势：**Join 查询性能提升 2-10 倍**（基于先进的 MPP 架构与基于成本的查询优化器）、**通过存算分离降低高达 70% 的基础设施成本**（支持计算与存储资源独立扩缩）、以及**卓越的实时更新性能**（写时合并引擎在高频数据更新时仍能保持稳定的查询性能）。


## 何时选择 Apache Doris

| 场景 | 推荐选择 | 原因 |
|------|----------|------|
| 需要复杂多表 Join | **Apache Doris** | MPP 架构原生支持分布式 Join，ClickHouse 依赖宽表建模 |
| 高并发查询 (100+ 并发) | **Apache Doris** | 支持数千并发，ClickHouse 通常少于 100 |
| 高频实时更新 + UPSERT | **Apache Doris** | 写时合并引擎保证查询性能，ClickHouse 合并开销大 |
| 主要做单表聚合查询 | 两者皆可 | 根据团队熟悉度选择 |
| 已有 ClickHouse 稳定运行 | 继续用 ClickHouse | 迁移有成本，收益需评估 |

## 性能对比

![性能对比](/images/next/doris-vs-clickhouse/benchmark.jpg)

### ClickBench

ClickBench 是专用于评估分析型数据库性能的基准测试工具，其核心验证场景聚焦于海量宽表处理效能。

**测试场景**：点击流分析、结构化日志处理等典型 OLAP 应用。

### TPC-H SF100

业界公认的数据库性能测试标准，包含 22 条模拟真实企业场景的复杂 SQL 查询。

:::note 注意
ClickHouse 在测试过程中存在 7 个未能完成的查询，最终测试的总耗时结果为 Doris 完整执行 22 个查询的时间、ClickHouse 仅为成功执行 15 个查询的时间。
:::

### TPC-DS 1TB

数据仓库与分析型数据库的权威性能测试，涵盖 99 条复杂查询，模拟网络销售、门店销售等真实业务场景。

:::note 注意
TPC-DS 性能测试中使用了大量关联子查询，而 ClickHouse 在测试时（2024 年 9 月）尚未支持该功能，导致约半数查询无法执行。
:::


## 常见问题 FAQ

### Q1: Doris 和 ClickHouse 的核心区别是什么？

Apache Doris 基于 MPP 架构提供完整的分布式 Join 能力，而 ClickHouse 采用 Scatter-Gather 架构，复杂多表关联需要依赖宽表建模。

- **Apache Doris**：MPP 架构下，复杂查询可跨节点分布式执行，查询优化器自动选择最优 Join 策略
- **ClickHouse**：Scatter-Gather 架构下，数据需要在各节点独立处理后汇总，无法原生执行跨节点分布式 Join

**适用场景建议**：
- 频繁多表关联 → 选择 Apache Doris
- 主要是单表聚合查询 → 两者皆可

---

### Q2: Doris 的 Join 性能真的比 ClickHouse 快吗？

**是的**。基于标准 TPC-H 和 TPC-DS 基准测试：

| 基准测试 | Apache Doris | ClickHouse |
|----------|-------------|------------|
| TPC-H SF100 | 22 个查询全部完成 | 7 个查询 OOM 失败 |
| TPC-DS | 99 个查询全部完成 | 约 50% 查询因不支持关联子查询失败 |

**原因**：Doris 的 CBO 查询优化器会自动选择最优 Join 策略（Broadcast/Shuffle/Colocate），而 ClickHouse 需要手动调优。

---

### Q3: 从 ClickHouse 迁移到 Doris 需要注意什么？

1. **SQL 语法差异**：Doris 兼容 MySQL 协议，ClickHouse 语法需调整
2. **数据模型**：ClickHouse 的宽表建模方式在 Doris 中不一定最优
3. **导入工具**：需要替换数据导入脚本，Doris 提供多种导入方式
4. **建议**：先在小规模场景验证 SQL 兼容性，再逐步迁移

---

### Q4: Doris 的存算分离和 ClickHouse Cloud 有什么区别？

| 特性 | Apache Doris | ClickHouse Cloud |
|------|-------------|------------------|
| 存算分离 | **开源版本已支持**（自 3.0） | 仅商业版 |
| 扩容行为 | 弹性计算，节点增减无需数据重平衡 | 扩容需数据重平衡 |
| 成本模型 | 按需付费，只为实际计算资源买单 | 按实例付费 |

---

### Q5: Doris 能否查询数据湖？

**能**。Apache Doris 内置数据湖查询能力：

- 支持直接查询 **Hive、Hudi、Iceberg、Parquet** 等格式
- 无需数据导入，即可实现湖仓一体
- 支持 Arrow-Flight 协议与其他引擎集成（如 AI/数据科学工具）
