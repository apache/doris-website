Vectorized execution improves CPU efficiency by processing data in columnar batches instead of row by row. That makes it a good fit for analytical workloads that need better throughput.

## When to use

Use it when query performance depends heavily on CPU efficiency and batch processing can reduce per-row overhead.

## Key points

- Batch processing usually improves cache locality.
- The benefit is strongest on scan-heavy workloads.
- It works well with distributed query execution.
