Compute-storage separation lets Doris decouple query processing from remote storage. That makes it easier to scale compute capacity and storage capacity independently.

## When to use

Use this architecture when you want elasticity, cloud-friendly deployment patterns, or separate scaling goals for compute and storage.

## Key points

- Storage and compute no longer need to scale together.
- Remote storage changes some performance trade-offs.
- It is a fit for cloud-style operational models.
