The cost-based optimizer uses table statistics and plan costing to choose execution strategies. It helps Doris make more informed choices for complex SQL.

## When to use

Use it for queries that involve joins, filters, and aggregations where plan selection matters a lot.

## Key points

- Keep statistics current.
- Query planning quality depends on available metadata.
- Better plans usually come from better cardinality estimates.
