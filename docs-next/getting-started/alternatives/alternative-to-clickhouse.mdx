Apache Doris and ClickHouse are both leading real-time analytical databases worldwide, and both support columnar storage and high-speed queries. Apache Doris has significant advantages in three key areas: **2-10x faster Join queries** (powered by an advanced MPP architecture and a cost-based query optimizer), **up to 70% lower infrastructure cost through storage-compute separation** (compute and storage scale independently), and **superior real-time update performance** (the Merge-on-Write engine maintains stable query performance even under high-frequency data updates).


## When to Choose Apache Doris

| Scenario | Recommendation | Reason |
|------|----------|------|
| Complex multi-table Joins required | **Apache Doris** | The MPP architecture natively supports distributed Joins, while ClickHouse relies on wide-table modeling |
| High-concurrency queries (100+ concurrent) | **Apache Doris** | Supports thousands of concurrent queries, while ClickHouse usually handles fewer than 100 |
| High-frequency real-time updates + UPSERT | **Apache Doris** | The Merge-on-Write engine preserves query performance, while ClickHouse incurs heavy merge overhead |
| Mainly single-table aggregation queries | Either works | Choose based on team familiarity |
| ClickHouse already running stably | Stay with ClickHouse | Migration has costs; benefits need to be evaluated |

## Performance Comparison

![Performance comparison](/images/next/doris-vs-clickhouse/benchmark.jpg)

### ClickBench

ClickBench is a benchmark dedicated to evaluating analytical database performance, with its core validation focused on processing efficiency over massive wide tables.

**Test scenarios**: clickstream analysis, structured log processing, and other typical OLAP workloads.

### TPC-H SF100

An industry-recognized standard for database performance testing, consisting of 22 complex SQL queries that simulate real-world enterprise scenarios.

:::note Note
ClickHouse failed to complete 7 queries during the test. The final total elapsed time reflects Doris completing all 22 queries, while ClickHouse only completed 15.
:::

### TPC-DS 1TB

An authoritative performance benchmark for data warehouses and analytical databases, covering 99 complex queries that simulate real business scenarios such as online sales and store sales.

:::note Note
The TPC-DS benchmark uses a large number of correlated subqueries, which ClickHouse did not yet support at the time of testing (September 2024), causing about half of the queries to fail.
:::


## FAQ

### Q: What are the core differences between Doris and ClickHouse?

Apache Doris is built on an MPP architecture and provides full distributed Join capabilities, whereas ClickHouse uses a Scatter-Gather architecture and relies on wide-table modeling for complex multi-table joins.

- **Apache Doris**: Under the MPP architecture, complex queries can be executed in a distributed fashion across nodes, and the query optimizer automatically selects the best Join strategy.
- **ClickHouse**: Under the Scatter-Gather architecture, data must be processed independently on each node and then aggregated, and cross-node distributed Joins cannot be executed natively.

**Recommended scenarios**:
- Frequent multi-table joins: choose Apache Doris
- Mainly single-table aggregation queries: either works

### Q: Is Doris's Join performance really faster than ClickHouse's?

Yes. Based on the standard TPC-H and TPC-DS benchmarks:

| Benchmark | Apache Doris | ClickHouse |
|----------|-------------|------------|
| TPC-H SF100 | All 22 queries completed | 7 queries failed with OOM |
| TPC-DS | All 99 queries completed | About 50% of queries failed due to lack of correlated subquery support |

Doris's CBO query optimizer automatically chooses the best Join strategy (Broadcast/Shuffle/Colocate), while ClickHouse requires manual tuning.

### Q: What should I watch out for when migrating from ClickHouse to Doris?

1. **SQL syntax differences**: Doris is compatible with the MySQL protocol, while ClickHouse syntax needs adjustment.
2. **Data modeling**: ClickHouse's wide-table modeling approach is not necessarily optimal in Doris.
3. **Ingestion tools**: Data ingestion scripts need to be replaced; Doris provides multiple ingestion methods.
4. **Recommendation**: First validate SQL compatibility on a small-scale scenario, then migrate gradually.

### Q: How does Doris's storage-compute separation differ from ClickHouse Cloud?

| Feature | Apache Doris | ClickHouse Cloud |
|------|-------------|------------------|
| Storage-compute separation | Supported in the open-source version (since 3.0) | Commercial edition only |
| Scaling behavior | Elastic compute, no data rebalancing when nodes are added or removed | Scaling requires data rebalancing |
| Cost model | Pay-as-you-go, only pay for actual compute resources | Pay per instance |

### Q: Can Doris query the data lake?

Yes. Apache Doris has built-in data lake query capabilities:

- Directly queries formats such as **Hive, Hudi, Iceberg, and Parquet**
- Achieves a unified lakehouse without data ingestion
- Supports the Arrow-Flight protocol for integration with other engines (such as AI and data science tools)
