Apache Doris and Trino/Presto are both mainstream lakehouse query engines, but Doris has a clear performance advantage. Trino/Presto focuses primarily on query acceleration, while Doris not only accelerates queries but also serves as a standalone data warehouse. Enterprises can use Doris to unify the data warehouse and lakehouse query engine, simplifying their data architecture.

- **Unified**: Apache Doris unifies the data warehouse and the lakehouse query engine, effectively simplifying the enterprise technology stack.

- **10x performance gains**: Compared with Presto/Trino, Doris delivers up to 10x faster internal table query performance.

- **2-3x execution efficiency**: As a lakehouse engine, Doris executes 2-3x faster than Presto/Trino.


## When to Choose Apache Doris

| Scenario | Recommended Choice | Reason |
|------|----------|------|
| High-concurrency real-time analytics (QPS > 1000) | **Apache Doris** | Supports thousands of concurrent queries with excellent real-time analytics performance |
| Unified data warehouse + lakehouse queries | **Apache Doris** | Integrated architecture that simplifies the technology stack |
| Maximum internal table query performance | **Apache Doris** | Fully vectorized engine delivers up to 10x performance gains |
| Transparent acceleration via automatic materialized views | **Apache Doris** | The query optimizer automatically matches the optimal materialized view |
| Federated query capability only | **Trino/Presto** | Strong at querying across heterogeneous data sources |
| Mature data warehouse already in place, no migration desired | **Trino/Presto** | No need to replace the existing architecture |

## Performance Comparison

### TPC-DS 1TB Benchmark

The TPC-DS 1TB benchmark uses a 1TB dataset with 24 tables and 6.35 billion records, evaluating data warehouse performance through 99 complex queries that cover correlated queries, aggregation, and nested subqueries. Built on a snowflake schema, the benchmark simulates a real-world e-commerce sales scenario, and the complexity of the queries makes processing 1TB of data a significant challenge.

**Test environment:**

- 1 FE/Coordinator node and 5 BE/Worker nodes
- 64-core CPU, 1.5TB memory, and SSD storage per node
- HDFS deployed on the compute nodes, with Hive tables created

**Test results:**

- When data is loaded into Doris internal tables and queried by Doris, overall query latency is the lowest.
- When Doris and Trino each query external Hive tables, Doris delivers better query acceleration in data lake scenarios.

## FAQ

### Q: Can Doris and Trino coexist?

In terms of functionality, Doris can replace Trino as long as the required data source types are supported. The actual decision depends on the migration plan and compatibility with existing workloads.

### Q: How long does migration to Doris take?

Doris provides a rich set of ETL tools and Hive-compatible APIs. Migration typically takes 2 to 4 weeks.

### Q: Is Doris really 2-3x faster than Trino for lakehouse queries?

Yes. According to the TPC-DS 1TB benchmark, Doris delivers better query acceleration than Trino in data lake scenarios when querying external Hive tables.

### Q: How do I choose the right materialized view strategy?

Doris supports multiple materialized view refresh strategies. Choose incremental refresh or full refresh based on data update frequency and query patterns. The query optimizer automatically matches the optimal materialized view.
