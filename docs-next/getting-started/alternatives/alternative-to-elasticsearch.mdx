Elasticsearch and Apache Doris are widely used in observability, cybersecurity, and real-time analytics. Elasticsearch incurs high overhead in storage resources and write costs. By contrast, Apache Doris significantly reduces resource consumption through its efficient storage architecture and high storage compression ratio, while providing more comprehensive analytical capabilities and better query performance.


## When to Choose Apache Doris

| Scenario | Recommended Choice | Reason |
|------|----------|------|
| Need lower storage costs | **Apache Doris** | Compression ratio of 1:5-1:10, far better than Elasticsearch's 1:1.5 |
| Need complex analytical capabilities | **Apache Doris** | Supports multi-table joins, materialized views, and standard SQL, with no need to learn a dedicated DSL |
| Need lower total cost of ownership | **Apache Doris** | Supports a separation of storage and compute architecture, with more flexible elastic scaling |
| Need higher write throughput | **Apache Doris** | Write performance overhead is below 10%, while Elasticsearch can be up to 3x |
| Mainly performing log search and simple aggregation | Either works | Choose based on team familiarity |
| Existing Elasticsearch running stably | Continue with Elasticsearch | Migration has costs, and the benefits need to be evaluated |

## FAQ

### Q: How to migrate from Elasticsearch to Apache Doris?

Apache Doris provides multiple data import solutions and supports full data migration from Elasticsearch. For specific steps, refer to the [data import documentation](../../data-operate/import/load-manual).

### Q: What is the learning curve for Doris?

Doris supports standard SQL and connects via the MySQL protocol. Engineers familiar with SQL do not need to learn an additional query language.

### Q: What is the difference in log search performance between Doris and Elasticsearch?

The two have similar performance in simple keyword search scenarios. However, Apache Doris performs better in aggregation analysis and time-series data analysis scenarios, and supports a richer set of analytical functions.
