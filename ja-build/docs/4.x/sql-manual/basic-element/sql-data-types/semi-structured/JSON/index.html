JSON

JSON Introduction​

JSON (JavaScript Object Notation) は、人間が読みやすいテキストを使用してデータを保存および送信するオープンスタンダードファイル形式およびデータ交換形式です。公式仕様 RFC7159 によると、JSONは以下の基本型をサポートしています：

  • Bool
  • Null
  • Number
  • String
  • Array
  • Object

JSON データ型は JSON データをバイナリ形式で効率的に格納し、JSON関数を通じてその内部フィールドへのアクセスを可能にします。

デフォルトでは、最大1048576バイト（1MB）をサポートし、最大2147483643バイト（2GB）まで拡張可能です。これは string_type_length_soft_limit_bytes 設定により調整できます。

通常の STRING 型でJSON文字列を格納する場合と比較して、JSON型には2つの主な利点があります：

  1. データ挿入時のJSON形式バリデーション
  2. より効率的なバイナリストレージ形式により、get_json_xx 関数と比較して、json_extract などの関数を使用したJSON内部フィールドへの高速アクセスが可能
Note

バージョン1.2.xでは、JSON型はJSONBと命名されていました。MySQLとの互換性を維持するため、バージョン2.0.0からJSONに名称変更されました。古いtableでは引き続き以前の名前を使用できます。

JSON Number精度の問題​

JSONを使用する際は、数値精度について特別な注意が必要です：

  • ほとんどのシステムでは、Number型はIEEE 754-2008 binary 64-bit（倍精度）浮動小数点数に基づいて実装されています（例：C++のdouble型）
  • JSON仕様ではNumberの基盤となる型を厳密に定義しておらず、JSONデータはテキストとして異なるシステム間で交換されるため、精度損失が発生する可能性があります

{"abc": 18446744073709551616} のようなJSON文字列の場合：

-- Conversion result in MySQL
cast('{"abc": 18446744073709551616}' as json)
-- Result: {"abc": 1.8446744073709552e19}
// Conversion result in JavaScript
console.log(JSON.parse('{"abc": 18446744073709551616}'));
// Result: {abc: 18446744073709552000}

システム間でデータを交換する際に数値の精度を保持するため、大きな数値は文字列として保存する必要があります。例：{"abc": "18446744073709551616"}

DorisにおけるJSON型​

DorisはJSON標準仕様に準拠したデータ型をサポートし、バイナリエンコーディング保存には効率的なJSONB（JSON Binary）形式を使用します。

サポートされる型​

Doris JSONBは標準的なJSON型をすべてサポートしています。主な違いは、DorisのNumber型に対してより細かい拡張を提供し、Dorisの内部型により正確にマッピングできることです。

JSON型サブタイプ対応するDoris型
Bool-BOOLEAN
Null-（直接的な等価型なし、JSON null値を表す）
NumberInt8TINYINT
Int16SMALLINT
Int32INT
Int64BIGINT
Int128LARGEINT
DoubleDOUBLE
FloatFLOAT
DecimalDECIMAL
String-STRING
Array-ARRAY
Object-STRUCT

重要な注意点：​

  • Nullの意味：
    • JSONのNullは「空の値」を表す有効な値です。これは「不明」または「欠損」を表すSQLのNULLとは異なります。
    • CAST('null' AS JSON)の結果は、JSON null値を含むJSONB列となり、それ自体はSQLレベルではNULLではありません。
    • CAST('null' AS JSON) IS NULLはfalse（0）を返します。これは、その列が既知のJSON null値を含んでおり、SQL NULLではないためです。

操作と制限事項​

  • 比較と算術演算：
    • JSONB列は他のデータ型（他のJSONB列を含む）と直接比較したり、算術演算に使用したりすることはできません。
    • 解決策：JSON_EXTRACT関数を使用してJSONBからスカラー値（INT、DOUBLE、STRING、BOOLEANなど）を抽出し、対応するネイティブなDoris型に変換して比較や計算を行います。
  • ソートとグループ化：
    • JSONB列はORDER BYおよびGROUP BY操作をサポートしていません。
  • 暗黙的な変換：
    • 入力のみ：JSONB列にデータを入力する際、STRING型はJSONBに暗黙的に変換できます（文字列の内容が有効なJSONテキストである場合）。他のDoris型はJSONBに暗黙的に変換できません。

構文​

定義：

json_column_name JSON

挿入:

  • INSERT INTO VALUESを使用し、フォーマットをクォートで囲んだ文字列として指定します。例:
INSERT INTO table_name(id, json_column_name) VALUES (1, '{"k1": "100"}')
  • STREAM LOADの場合、対応する列の形式は追加の引用符なしの文字列です。例：
12	{"k1":"v31", "k2": 300}
13	[]
14	[123, 456]
  • JSON内に'\r'、'\t'などの'\'を含む特殊文字が現れる場合、replace関数を使用して"\"を"\\"に置換する必要があります。例えば、"\n"を"\\n"に置換する必要があります。

Query:

  • JSON列全体を直接選択する場合：
SELECT json_column_name FROM table_name;
  • JSON関数を使用してJSONから特定のフィールドやその他の情報を抽出します。例：
SELECT json_extract(json_column_name, '$.k1') FROM table_name;

JSON型は、integer、string、BOOLEAN、ARRAY、MAPとの間でキャストできます。例：

SELECT CAST('{"k1": "100"}' AS JSON);
SELECT CAST(json_column_name AS STRING) FROM table_name;
SELECT CAST(json_extract(json_column_name, '$.k1') AS INT) FROM table_name;
ヒント

JSON型は現在、GROUP BY、ORDER BY、または比較演算では使用できません。

JSON Input​

JSON構文に準拠する文字列をCASTを使用してJSONBに変換します。

-- Simple scalar/basic values (numeric types, bool, null, string)
mysql> SELECT cast('5' as json);
+-------------------+
| cast('5' as json) |
+-------------------+
| 5                 |
+-------------------+

-- Arrays with zero or more elements (elements don't need to be the same type)
mysql> SELECT cast('[1, 2, "foo", null]' as json);
+-------------------------------------+
| cast('[1, 2, "foo", null]' as json) |
+-------------------------------------+
| [1,2,"foo",null]                    |
+-------------------------------------+

-- Objects containing key-value pairs
-- Note that object keys must always be quoted strings
mysql> SELECT cast('{"bar": "baz", "balance": 7.77, "active": false}' as json);
+------------------------------------------------------------------+
| cast('{"bar": "baz", "balance": 7.77, "active": false}' as json) |
+------------------------------------------------------------------+
| {"bar":"baz","balance":7.77,"active":false}                      |
+------------------------------------------------------------------+

-- Arrays and objects can be nested arbitrarily
mysql> SELECT cast('{"foo": [true, "bar"], "tags": {"a": 1, "b": null}}' as json);
+---------------------------------------------------------------------+
| cast('{"foo": [true, "bar"], "tags": {"a": 1, "b": null}}' as json) |
+---------------------------------------------------------------------+
| {"foo":[true,"bar"],"tags":{"a":1,"b":null}}                        |
+---------------------------------------------------------------------+

DorisのJSONBは、空白文字などの意味的に無関係な詳細情報を保持しません。

mysql> -- The input text and JSON output may not look the same
mysql> SELECT cast('[1,                 2]' as json);
+----------------------------------------+
| cast('[1,                 2]' as json) |
+----------------------------------------+
| [1,2]                                  |
+----------------------------------------+

主な相違点と注意事項：​

  • CAST(string AS JSON): JSON構文に準拠した文字列を解析するために使用されます。
  • CAST(string AS JSON): Number型の場合、Int8、Int16、Int32、Int64、Int128、Double型のみを解析し、Decimal型は解析しません。
  • 他のほとんどのJSON実装とは異なり、DorisのJSONB型はInt128精度まで対応しています。Int128精度を超える数値はオーバーフローする可能性があります。
  • 入力される数値文字列が12.34の場合、Doubleとして解析されます。小数点がない場合は整数として解析されます（サイズがInt128の範囲を超える場合は、Doubleに変換されますが精度が失われます）

to_jsonを使用してDoris内部型をJSONB型に変換する​

mysql> SELECT to_json(1) , to_json(3.14) , to_json("12345");
+------------+---------------+------------------+
| to_json(1) | to_json(3.14) | to_json("12345") |
+------------+---------------+------------------+
| 1          | 3.14          | "12345"          |
+------------+---------------+------------------+

mysql> SELECT to_json(array(array(1,2,3),array(4,5,6)));
+-------------------------------------------+
| to_json(array(array(1,2,3),array(4,5,6))) |
+-------------------------------------------+
| [[1,2,3],[4,5,6]]                         |
+-------------------------------------------+

mysql> SELECT json_extract(to_json(array(array(1,2,3),array(4,5,6))), '$.[1].[2]');
+----------------------------------------------------------------------+
| json_extract(to_json(array(array(1,2,3),array(4,5,6))), '$.[1].[2]') |
+----------------------------------------------------------------------+
| 6                                                                    |
+----------------------------------------------------------------------+

mysql> SELECT to_json(struct(123,array(4,5,6),"789"));
+------------------------------------------+
| to_json(struct(123,array(4,5,6),"789"))  |
+------------------------------------------+
| {"col1":123,"col2":[4,5,6],"col3":"789"} |
+------------------------------------------+

mysql> SELECT json_extract(to_json(struct(123,array(4,5,6),"789")),"$.col2");
+----------------------------------------------------------------+
| json_extract(to_json(struct(123,array(4,5,6),"789")),"$.col2") |
+----------------------------------------------------------------+
| [4,5,6]                                                        |
+----------------------------------------------------------------+

to_jsonはJSONB型にマッピングされるDoris型の変換のみをサポートします。 例えば、DECIMALはto_jsonで使用できます。 しかし、DATEは使用できません。まずSTRINGに変換してから、to_jsonを使用する必要があります。

JSONB出力​

他のシステムとの連携のためにプレーンテキストに変換する際、DorisのJSONB型は有効なJSONテキストの生成を保証します：

  1. Null値：
    • null（引用符なし）として出力
  2. Boolean値：
    • true → trueを出力
    • false → falseを出力
  3. 数値型：
    • すべての数値は直接出力
    • 例：5 → 5を出力、3.14 → 3.14を出力
  4. 文字列：
    • 二重引用符で出力："<内容>"
    • 特殊文字はエスケープされます：
      • " → "
      • \ → \
      • / → /
      • Backspace → \b
      • Form feed → \f
      • Newline → \n
      • Carriage return → \r
      • Tab → \t
    • その他の制御文字（ASCII < 32）はUnicodeエスケープシーケンスに変換：\uXXXX
  5. オブジェクト：
    • 形式：{<キー・バリューペアのリスト>}
    • キー・バリューペアの形式："<キー>": <値>
    • 複数のキー・バリューペアはカンマで区切り
  6. 配列：
    • 形式：[<要素のリスト>]
    • 複数の要素はカンマで区切り
  7. ネストした構造の処理：
    • オブジェクトと配列は無制限のネストレベルをサポート
    • 各ネストレベルは同じルールを使用して再帰的に処理

数値精度の問題​

to_jsonを使用してDoris内部型をJSONBに変換する場合、精度の損失は発生しません。 Doris内部のJSON関数を使用する際、戻り値もJSONB型の場合、精度の損失は発生しません。 しかし、DorisのJSONBをプレーンテキストに変換してからJSONBに戻すと、精度の損失が発生する可能性があります。

例：Doris JSON型オブジェクト

Object{
    "a": (Decimal 18446744073709551616.123)
}

Converted to plain text:

{"a": 18446744073709551616.123}

プレーンテキストがDoris JSON型に変換される場合：

Object{
    "a": (Double 18446744073709552000)  // precision loss
}

設定と制限事項​

  • JSONはデフォルトで1,048,576バイト（1 MB）をサポートします
  • サイズ制限はBE設定パラメータstring_type_length_soft_limit_bytesで調整できます
  • 最大2,147,483,643バイト（約2 GB）まで調整可能です
  • DorisのJSONタイプObjectsでは、キー長は255バイトを超えることはできません

FAQ​

  1. JSONのnullとSQLのNULL（つまり、IS NULL）に違いはありますか？

はい、違いがあります。JSONでは、null（例：{"key1": null}）はキーkey1が存在し、その値が明示的にnullであることを意味します。これはJSONバイナリにエンコードされる特別な型です。

対照的に、SQL NULL（IS NULLを使用する場合）は、キーがJSONオブジェクト内に全く存在しないことを示す場合があります。

例：

mysql> SELECT JSON_EXTRACT_STRING('{"key1" : null}', "$.key1") IS NULL;
+----------------------------------------------------------+
| JSON_EXTRACT_STRING('{"key1" : null}', "$.key1") IS NULL |
+----------------------------------------------------------+
|                                                        0 |
+----------------------------------------------------------+
1 row in set (0.00 sec)

mysql> SELECT JSON_EXTRACT_STRING('{"key1" : null}', "$.key_not_exist") IS NULL;
+-------------------------------------------------------------------+
| JSON_EXTRACT_STRING('{"key1" : null}', "$.key_not_exist") IS NULL |
+-------------------------------------------------------------------+
|                                                                 1 |
+-------------------------------------------------------------------+
1 row in set (0.01 sec)
  1. GET_JSON_XXXとJSON_EXTRACT_XXX関数の違いは何ですか？どちらを選択すべきでしょうか？

GET_JSON_XXX関数は文字列型での使用を想定して設計されており、生のJSON文字列から直接値を抽出します。一方、JSON_EXTRACT_XXX関数はJSONデータ型専用に実装されており、そのデータ型に対して最適化されています。
