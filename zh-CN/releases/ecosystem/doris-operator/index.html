Doris Operator

本文按版本倒序列出 Doris Operator 的版本发布说明。

26.0.0​

来源：Release Notes 26.0.0

该版本改进了 DDC 生命周期管理与可靠性，包括计算组优雅停机滚动、可配置的就绪探针策略、DCR mTLS 支持以及更清晰的 DDC 健康状态上报。同时，该版本增强了 Helm chart 配置，并增加了多项校验和稳定性修复。

功能与改进​

  • DDC 计算组在滚动更新、缩容和删除操作中支持优雅停机滚动，提供 sentinel 兼容性开关，并在镜像不支持时回退到常规 StatefulSet 流程。#502
  • DorisCluster 和 DorisDisaggregatedCluster Pod 支持可配置的就绪探针策略。#486
  • DCR 支持 mTLS。#483
  • 将 DDC 计算组 Service 拆分为内部 headless Service 和外部 Service，使升级和流量切换更平滑。#484
  • Helm chart 增加组件级 Secret 支持。#465
  • Helm chart 暴露已部署 Pod 的 serviceAccount 配置。#470

Bug 修复​

  • 修复 DDC MetaService 健康状态聚合，使 MetaService 降级能够反映到集群健康状态中。#500
  • 修复 dcr.Annotations 为 nil 时 CompareConfigmapAndTriggerRestart 中的 nil map panic。#493
  • 修复 FE endpoint 地址的 JSON tag，并重新生成 DorisCluster CRD。#497
  • 拒绝 feSpec.replicas < feSpec.electionNumber 的非法 DorisDisaggregatedCluster FE 配置。#498
  • 在应用不可变的 StatefulSet volume claim template 之前，拒绝 DDC 计算组存储模板变更。#496
  • 修复 DorisDisaggregatedCluster BE Helm 配置中的默认 storage_root_path。#477

可靠性​

  • 拒绝在 DorisCluster 和 DorisDisaggregatedCluster webhook 中使用内置 admin 账户作为 Operator 管理用户。#499
  • 增加针对 Go 格式化、vet/lint 检查、Operator 构建以及生成的 manifest 和 artifact 的 Pull Request 检查。#501

致谢​

  • ecniiv
  • jonasbrami
  • catpineapple
  • JinVei
  • Al-assad

25.8.0​

来源：Release Notes 25.8.0

该版本为 DDC 增加了重要能力，包括 TLS 支持、PVC 扩容和 Pod 调试能力。同时，该版本改进了 Helm 支持、Pod 资源配置和可靠性。

功能与改进​

  • dorisctl 支持 TLS 访问。#432
  • Operator 在 drop node 场景中支持使用 TLS。#434
  • 改进 Helm chart 模板。#452
  • Helm 和 Operator chart 支持通过 values.yaml 设置 Deployment 资源配额。#462
  • Helm chart kubeVersion 兼容带预发布后缀的 Kubernetes 版本，例如 EKS 和 GKE 版本。#463
  • 增加 DDC 调试能力。#460 #471
  • 为 BE Pod 增加 CPU 资源感知参数传递能力。#464
  • DDC 支持 PVC 扩容。#467
  • Init Container 自定义行为支持资源限制配置。#469

Bug 修复​

  • 修复单元测试用例问题。#430 #433
  • 修复未关闭 SQL 连接导致的资源泄漏。#437
  • 修复 FE PVC 信息收集问题。#438
  • 修复错误日志信息、重复环境变量和其他代码缺陷。#440 #442 #449
  • 修复 CVE 并更新安全依赖。#453
  • 修复 MySQL SQL Client 结构定义映射问题。#468

可靠性​

  • DDC MetaService controller 复用 BuildVolumesVolumeMountsAndPVCs 构建 PVC，提升逻辑一致性。#436

下载​

镜像格式请参考镜像仓库说明。

致谢​

  • ztonny
  • matiasbertani
  • jonasbrami
  • gohalo
  • catpineapple

  • 增加默认 Issue 模板。#288

Bug 修复​

  • 修复 DorisCluster 资源中未设置 FE electionNumber 时的空指针错误。#285
  • 修复 FE 访问地址，改为使用 Service 名称和 namespace。#291

24.0.0​

来源：Release Note 24.0.0

该版本引入 DorisDisaggregatedCluster（DDC）自定义资源，用于部署存算分离 Apache Doris 集群。同时，该版本通过 DorisCluster（DCR）扩展存算一体部署能力。

DDC 仅适用于 Apache Doris 3.0.2 或更高版本。

新功能​

DorisDisaggregatedCluster​

  • 增加 FoundationDB 模块访问能力。#186
  • 支持部署 MetaService 模块。#194
  • 支持部署 FE 模块。#199
  • 支持部署计算组模块。#185 #189 #192 #197
  • 为存算分离 Doris 组件增加镜像 entrypoint 支持。#195 #196
  • 支持缩容 FE。#225
  • 支持缩容计算组并清理资源。#238
  • 支持通过 SQL 接口缩容 FE。#255
  • 支持通过 SQL 接口缩容计算组。#256

DorisCluster​

  • 通过 External Service 暴露 Arrow Flight SQL 端口。#251

功能改进​

DorisDisaggregatedCluster​

  • 支持通过 be.conf 和 PVC 配置创建计算组。#198
  • 改进 README 和 DDC 示例 YAML 文件。#203 #210 #211 #214
  • 将 README 示例更新为使用最新版本。#220
  • 增加用于启用 DDC 部署的 Operator 开关。#204
  • 增加 SystemInitialization，用于准备系统环境。#212
  • 根据 Doris 命名变更同步计算组术语。#215 #245
  • 删除 MetaService 副本 CRD。#227
  • 将 MetaService CRD 重构为 DorisDisaggregatedCluster CRD。#234
  • 将计算组标识符改为新 SQL 接口使用的 UniqueId。#239
  • 增加 FE SQL 镜像 entrypoint 和集群 ID 哈希值。#249
  • 将 DDC 操作从 HTTP 接口迁移到 SQL 接口。#254
  • 为 Pod liveness probe 增加 timeoutSeconds。#257
  • 增加 logNotStore，使 FE 可以跳过创建日志 PVC。#266
  • 更新 DDC 示例和文档。#268

DorisCluster​

  • 为 Pod liveness probe 增加 timeoutSeconds。#257

其他变更​

  • 增加公共工具函数单元测试。#226
  • 增加 resource model 和 controller model 单元测试。#232
  • 更新用于 Artifact Hub 的 Helm chart 配置。#244
  • 将组织名称从 SelectDB 改为 Apache。#247

Bug 修复​

DorisDisaggregatedCluster​

  • 修复 Pod affinity 和 StatefulSet PVC 指针处理问题。#209
  • 修正 BE 默认存储路径。#243
  • 修复首次部署期间重复的 ms_token 环境变量导致 Pod 重启的问题。#259
  • 修复镜像 entrypoint 使用的 BE ConfigMap 路径。#261
  • 修复 electionNumber 为 nil 时的部署问题，并避免 Service 持续更新。#262 #266
  • 修复重复 reconcile 问题。#263 #265
  • 修正 BE 默认缓存路径。#266

DorisCluster​

  • 避免升级 Operator 时重启 Doris Pod。#226
  • 合并共享 Pod 环境变量数组时使用深拷贝。#236
  • 修复跨 namespace 缩容 FE 时 MySQL Client 失败的问题。#240
  • 修复 electionNumber 为 nil 时的部署失败问题。#260

其他变更​

  • 修复 StatefulSet 的 resource model 单元测试。#252

致谢​

  • intelligentfu8
  • catpineapple
  • hechao-ustc
