## 説明

json文字列内の指定されたパス内の整数（BIGINT）コンテンツを解析して取得します。

## 構文

```sql
GET_JSON_BIGINT( <json_str>, <json_path>)
```
## 必須パラメータ
| パラメータ | 説明 |
|------|------|
| `<json_str>`| データを抽出するJSONストリングが必要です。 |
| `<json_path>`| JSONパス、フィールドの場所を指定します。パスはドット記法で表記できます。 |


## 戻り値
- パスが指し示すフィールドのBIGINT値を返します。
- 指定されたパスに対応するフィールドが見つからない場合、またはフィールド値をBIGINT型に変換できない場合はNULLを返します。

## 使用上の注意

`<json_str>`は$記号で始まり、.をパス区切り文字として使用する必要があります。パスに.が含まれる場合は、ダブルクォートで囲むことができます。
[ ]を使用して配列インデックスを表し、0から開始します。
pathの内容には"、[、]を含めることはできません。
`<json_str>`の形式が正しくない場合、または`<json_path>`の形式が正しくない場合、または一致が見つからない場合はNULLを返します。
また、同じ機能を実現するためにはjsonb型とjsonb_extract_XXX関数の使用を推奨します。
特殊な状況は以下のように処理されます：
- JSONに指定されたフィールドが存在しない場合は`<json_path>`NULLを返します
- JSON内で`<json_path>`が指定するフィールドの実際の型がjson_extract_tで指定された型と一致しない場合、指定された型tに無損失で変換できる場合は指定された型tが返され、変換できない場合はNULLが返されます。

## 例

1. キーが"k1"の値を取得

```sql
SELECT get_json_bigint('{"k1":1, "k2":"2"}', "$.k1");
```
```sql
+-----------------------------------------------+
| get_json_bigint('{"k1":1, "k2":"2"}', '$.k1') |
+-----------------------------------------------+
|                                             1 |
+-----------------------------------------------+
```
2. キー "my.key" を持つ配列の2番目の要素を取得します

```sql
SELECT get_json_bigint('{"k1":"v1", "my.key":[1, 1678708107000, 3]}', '$. "my.key"[1]');
```
```sql
+---------------------------------------------------------------------------------+
| get_json_bigint('{"k1":"v1", "my.key":[1, 1678708107000, 3]}', '$. "my.key"[1]') |
+---------------------------------------------------------------------------------+
|                                                                   1678708107000 |
+---------------------------------------------------------------------------------+
```
3. secondary path k1.key -&gt; k2 を使用して配列の最初の要素を取得します。

```sql
SELECT get_json_bigint('{"k1.key":{"k2":[1678708107000, 2]}}', '$. "k1.key".k2[0]');
```
```sql

+-----------------------------------------------------------------------------+
| get_json_bigint('{"k1.key":{"k2":[1678708107000, 2]}}', '$."k1.key".k2[0]') |
+-----------------------------------------------------------------------------+
|                                                               1678708107000 |
+-----------------------------------------------------------------------------+

```
