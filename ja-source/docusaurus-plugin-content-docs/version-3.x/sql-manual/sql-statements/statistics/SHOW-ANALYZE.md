## 説明

このステートメントは、統計収集ジョブのステータスを表示するために使用されます。

## 構文

```SQL
SHOW [AUTO] ANALYZE [ <table_name> | <job_id> ]
    [ WHERE STATE = {"PENDING" | "RUNNING" | "FINISHED" | "FAILED"} ];
```
## 必須パラメータ

**なし**

## オプションパラメータ

**1. `AUTO`**

> 自動ジョブに関する情報を表示します。指定されていない場合、デフォルトで手動ジョブに関する情報が表示されます。

**2. `<table_name>`**

> Table名。指定後、このTableに対応するジョブ情報を表示できます。指定されていない場合、デフォルトですべてのTableのジョブ情報が返されます。

**3. `<job_id>`**

> 統計ジョブID。ANALYZEで非同期収集を実行する際に取得されます。IDが指定されていない場合、このコマンドはすべてのジョブに関する情報を返します。

**4. `WHERE STATE = {"PENDING" | "RUNNING" | "FINISHED" | "FAILED"}`**

> ジョブ状態のフィルタ条件。指定されていない場合、デフォルトですべての状態のジョブに関する情報が表示されます。

## 戻り値

| Column | Note           |
| -- |--------------|
| job_id | 一意の統計ジョブid           |
| catalog_name |   Catalog名         |
| db_name | データベース名           |
| tbl_name | Table名         |
| col_name | カラム名リスト           |
| job_type |   ジョブタイプ           |
| analysis_type |  分析タイプ           |
| message | エラーメッセージ         |
| last_exec_time_in_ms | 最後の分析時間           |
| state |   ジョブ状態          |
| progress | ジョブ進行状況           |
| schedule_type | スケジュールタイプ         |
| start_time | ジョブ開始時間          |
| end_time |   ジョブ終了時間           |
| priority | ジョブ優先度           |
| enable_partition | パーティション収集有効フラグ         |

## アクセス制御要件

このSQLを実行するユーザーは、少なくとも以下の権限を持つ必要があります：

| Privilege | Object | 注釈                                    |
|:--------------| :------------- |:------------------------------------------------|
| SELECT_PRIV   | Table    | SHOWを実行する際、クエリ対象TableのSELECT_PRIV権限が必要です。 |
