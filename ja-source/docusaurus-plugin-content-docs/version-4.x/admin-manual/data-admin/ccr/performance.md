|--------------------|-------------------------------------------|---------------------------------------------|

## 増分同期パフォーマンステスト

### テスト手順

1. アップストリームクラスタでTPC-H 1Tのデータベースとテーブルを作成する。
2. TPC-H 1Tデータベースの同期ジョブを作成する。
3. TPC-H 1Tデータのインポート完了を待機し、完了時刻を記録する。
4. ダウンストリームデータ同期の完了を待機し、完了時刻を記録する。

### テスト結論
増分同期時間差: 33秒

---

## フル同期パフォーマンステスト

### テスト手順
1. アップストリームクラスタでTPC-H 1Tのデータベースとテーブル情報を作成し、データインポートを完了して完了時刻を記録する。
2. TPC-H 1Tデータベースの同期ジョブを作成する。
3. ダウンストリームデータ同期の完了を待機し、完了時刻を記録する。

### テスト結論
フル同期時間差: 6分1秒

---

## Flink同期パフォーマンステスト

### テスト手順
1. Flinkを使用してアップストリームで100,000,000件のレコードをインポートする。
2. ライブラリとテーブルの同期ジョブを作成する。
3. 各段階（例：1,000,000件、2,000,000件など）でダウンストリーム同期完了時刻とアップストリームインポート完了時刻の差を観察する。
4. アップストリームでの最後のインポート完了時刻を記録する。
5. ダウンストリーム同期完了時刻を記録する。

### テスト結論
各段階での**ラグ時間**は`5秒`以内に維持されています。
