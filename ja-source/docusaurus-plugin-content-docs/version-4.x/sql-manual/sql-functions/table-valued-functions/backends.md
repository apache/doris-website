## 説明

table関数はバックエンドの一時tableを生成し、現在のdorisクラスター内のBEノード情報を表示することができます。

## 構文

```sql
BACKENDS()
```
## Access Control Requirements

| Privilege  | Object | 注釈 |
| :--------- |:-------|:------|
| ADMIN_PRIV | global |       |

## 戻り値
| フィールド                   | 説明                                                                                                            |
|-----------------------------|----------------------------------------------------------------------------------------------------------------|
| **BackendId**               | 各バックエンドノードの一意識別子。                                                                                |
| **Host**                    | バックエンドノードのIPアドレスまたはホスト名。                                                                      |
| **HeartbeatPort**           | ヘルスチェック（heartbeat）に使用されるポート。                                                                   |
| **BePort**                  | クエリ実行のためのバックエンドノードとクラスター間の通信に使用されるポート。                                        |
| **HttpPort**                | バックエンドノードのHTTPポート。                                                                                  |
| **BrpcPort**                | BRPC通信に使用されるポート。                                                                                     |
| **ArrowFlightSqlPort**      | Arrow Flight SQLポート（高性能データ転送のためのApache Arrowとの統合に使用）。                                   |
| **LastStartTime**           | バックエンドノードの最後の開始時刻のタイムスタンプ。                                                               |
| **LastHeartbeat**           | バックエンドノードから受信した最後のheartbeatのタイムスタンプ。                                                    |
| **Alive**                   | バックエンドノードが生きているかどうか（true/false）。                                                            |
| **SystemDecommissioned**    | バックエンドノードが廃止されているかどうか。                                                                       |
| **TabletNum**               | バックエンドノードが管理するtabletの数。                                                                          |
| **DataUsedCapacity**        | バックエンドノードが使用しているディスク容量（MB単位）。                                                           |
| **TrashUsedCapacity**       | trashが使用しているディスク容量（MB単位）。                                                                       |
| **AvailCapacity**           | バックエンドノードの利用可能なディスク容量。                                                                       |
| **TotalCapacity**           | バックエンドノードの総ディスク容量。                                                                              |
| **UsedPct**                 | バックエンドノードが使用しているディスク容量の割合。                                                               |
| **MaxDiskUsedPct**          | 全tablet間での最大ディスク使用率。                                                                                |
| **RemoteUsedCapacity**      | リモートストレージが使用しているディスク容量（該当する場合）。                                                      |
| **Tag**                     | バックエンドノードに関連付けられたタグ、通常は分類に使用される（例：location）。                                    |
| **ErrMsg**                  | バックエンドノードから報告されたエラーメッセージ。                                                                 |
| **Version**                 | バックエンドノードのバージョン。                                                                                  |
| **Status**                  | バックエンドノードの現在のステータス、tabletの成功/失敗レポート、ロード時間、クエリステータスを含む。                |
| **HeartbeatFailureCounter** | heartbeat失敗の回数（ある場合）。                                                                                 |
| **NodeRole**                | バックエンドノードの役割。`mix`などがあり、これはノードがストレージとクエリ処理の両方を処理することを意味する。        |
| **CpuCores**                | バックエンドノードのCPUコア数。                                                                                   |
| **Memory**                  | バックエンドノードのメモリ容量。                                                                                  |
