# IPV4_NUM_TO_STRING

## デスクリプション
Int16、Int32、Int64の数値を受け取ります。これをビッグエンディアンのIPv4アドレスとして解釈します。対応するIPv4アドレスをA.B.C.D形式（10進数をドット区切りにした形式）の文字列として返します。

## Alias
- INET_NTOA

## Syntax

```sql
IPV4_NUM_TO_STRING(<ipv4_num>)
```
## パラメータ
| Parameter | デスクリプション                                      |
|-----------|--------------------------------------------------|
| `<ipv4_num>`      | ipv4から変換されたInt型  |


## Return Value
A.B.C.D形式（10進数のドット区切り）で対応するIPv4アドレスを含む文字列を返します。特殊なケース：
- 入力パラメータが負の値または`4294967295`（`'255.255.255.255'`の数値）より大きい場合、`NULL`を返します

## Example

```sql
select ipv4_num_to_string(3232235521);
```
```text
+--------------------------------+
| ipv4_num_to_string(3232235521) |
+--------------------------------+
| 192.168.0.1                    |
+--------------------------------+
```
```sql
select num,ipv4_num_to_string(num) from ipv4_bi;
```
```text
+------------+---------------------------+
| num        | ipv4_num_to_string(`num`) |
+------------+---------------------------+
|         -1 | NULL                      |
|          0 | 0.0.0.0                   |
| 2130706433 | 127.0.0.1                 |
| 4294967295 | 255.255.255.255           |
| 4294967296 | NULL                      |
+------------+---------------------------+
```
