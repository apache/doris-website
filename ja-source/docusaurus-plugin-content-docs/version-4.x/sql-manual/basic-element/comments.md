## 説明

コメントを使用することで、アプリケーションの可読性と保守性を向上させることができます。例えば、ステートメント内にコメントを含めて、アプリケーション内でのそのステートメントの目的を説明することができます。SQLステートメント内のコメント（HINTを除く）は、ステートメントの実行に影響しません。HINTでのこの特定の形式のコメントの使用については、[HINT](../../query-acceleration/hints/hints-overview.md)セクションを参照してください。

SQLステートメントでは、コメントは任意のキーワード、パラメータ、句読点の間に記述できます。ステートメント内にコメントを含める方法は2つあります：

- 複数行コメント：スラッシュとアスタリスク（`/*`）でコメントを開始し、その後にコメントテキストを続けます。このテキストは複数行にまたがることができます。アスタリスクとスラッシュ（`*/`）でコメントを終了します。開始文字と終了文字、およびテキストの間にスペースや改行は必要ありません。
- 単一行コメント：2つのダッシュ（`--`）でコメントを開始し、その後にコメントテキストを続けます。このテキストは新しい行に拡張することはできません。コメントは改行文字で終了します。

## 例

### 複数行コメント

```sql
SELECT /* This is a multi-line comment
          that can span multiple lines */
       column_name
FROM   table_name;
```
### 単行コメント

```sql
SELECT column_name -- This is a single-line comment
FROM   table_name;
```
