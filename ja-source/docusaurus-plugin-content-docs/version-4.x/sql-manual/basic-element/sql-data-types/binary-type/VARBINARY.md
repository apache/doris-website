## VARBINARY
### 説明
可変長のバイナリバイトシーケンスで、Mは最大長（バイト単位）を表します。

VARCHARとは異なり、文字セットやコレーションを持たずにデータをバイト単位で格納・比較するため、任意のバイナリデータ（例：ファイルフラグメント、ハッシュ、暗号化/圧縮データ）に適しており、最大長は2,147,483,647です。

現在、VARBINARYカラムは作成されたマテリアライズドビューでサポートされておらず、Group By keys、Join Keys、および比較述語での使用もサポートされていません。

バージョンと制限事項：4.0以降でサポート。現在、Dorisでのtable作成とストレージのカラムタイプとしてはサポートされていません。外部システムのBINARY/VARBINARYフィールドをCatalogsを介してDoris VARBINARYにマッピングしてクエリに使用することができます。
