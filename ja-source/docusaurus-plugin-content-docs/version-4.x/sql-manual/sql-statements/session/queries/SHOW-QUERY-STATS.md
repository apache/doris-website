## デスクリプション

このステートメントは、履歴クエリによってヒットしたデータベースtableカラムを表示するために使用されます

## Syntax

```sql
SHOW QUERY STATS [ { [FOR <db_name>] | [FROM <table_name>] } ] [ALL] [VERBOSE]];
```
## オプションパラメータ

**1. `<db_name>`**

> このパラメータが設定されている場合、データベースのマッチング情報が表示されます

**2. `<table_name>`**

> このパラメータが設定されている場合、Tableのマッチング状態が照会されます

**3. `ALL`**

> ALL すべてのインデックスのマッチング情報を表示するかどうかを指定します

**4. `VERBOSE`**

> VERBOSE 詳細なマッチング情報を表示します

## アクセス制御要件

このSQLコマンドを実行するユーザーは、少なくとも以下の権限を持っている必要があります：

| Privilege          | Object    | 注釈              |
|:-------------------|:----------|:-------------------|
| SELECT_PRIV        | DATABASE  | 照会するデータベースに対するSELECT権限を持っている必要があります |

## 使用上の注意

- データベースとTableの履歴クエリマッチング状態を照会します。データはfeの再起動後にリセットされ、統計は各feごとに収集されます。

- FOR DATABASEとFROM TABLEを使用して、照会するデータベースまたはTableのマッチング情報を指定し、その後にデータベース名またはTable名を続けることができます。

- ALLとVERBOSEパラメータは単独で使用することも組み合わせて使用することもできますが、最後に使用する必要があり、Table照会でのみ使用できます。

- 使用中のデータベースがない場合、単純に'SHOW QUERY STATS'を実行するとすべてのデータベースのヒットが表示されます。

- ヒット結果には2つの列がある場合があります：QueryCountは列が照会された回数を示し、FilterCountは列がwhere条件として照会された回数を示します。
