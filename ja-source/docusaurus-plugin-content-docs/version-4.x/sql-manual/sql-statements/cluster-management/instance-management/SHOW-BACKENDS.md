## デスクリプション

このステートメントはBEノードの基本ステータス情報を表示するために使用されます。

## Syntax

```sql
 SHOW BACKENDS
```
## 戻り値

| カラム                      | 備考                                                                                                                                                                                                                                                                                                                         |
|-------------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| BackendId               | 現在のBEのID。                                                                                                                                                                                                                                                                                                                    |
| Host                    | 現在のBEのIPアドレスまたはホスト名。                                                                                                                                                                                                                                                                               |
| HeartbeatPort           | 現在のBEのハートビートサービスの通信ポート。                                                                                                                                                                                                                                                                           |
| BePort                  | 現在のBEのthrift RPC通信ポート。                                                                                                                                                                                                                                                                         |
| HttpPort                | 現在のBEのHTTP通信ポート。                                                                                                                                                                                                                                                                               |
| BrpcPort                | 現在のBEのbRPC通信ポート。                                                                                                                                                                                                                                                                               |
| ArrowFlightSqlPort      | 現在のBEのArrowFlightプロトコルの通信ポート。                                                                                                                                                                                                                                                                        |
| LastStartTime           | 現在のBEが開始されたタイムスタンプ。                                                                                                                                                                                                                                                                                   |
| LastHeartbeat           | 現在のBEが送信した最後の成功したハートビートのタイムスタンプ。                                                                                                                                                                                                                                                       |
| Alive                   | 現在のBEが生存しているかどうか。                                                                                                                                                                                                                                                                             |
| SystemDecommissioned    | この値がtrueの場合、現在のBEノードが安全なデコミッション処理中であることを意味する。                                                                                                                                                                                                                                        |
| TabletNum               | 現在のBEに格納されているタブレットの数。                                                                                                                                                                                                                                                                              |
| DataUsedCapacity        | 現在のBEのデータが占有するディスク容量。                                                                                                                                                                                                                                                                       |
| TrashUsedCapacity       | 現在のBEのトラッシュ内のデータが占有するディスク容量。                                                                                                                                                                                                                                                          |
| AvailCapacity           | 現在のBEの利用可能なディスク容量。                                                                                                                                                                                                                                                                                  |
| TotalCapacity           | 現在のBEの総ディスク容量。TotalCapacity = AvailCapacity + TrashUsedCapacity + DataUsedCapacity + その他の非ユーザーデータファイルが占有するディスク容量。                                                                                                                                                             |
| UsedPct                 | 現在のBEの総使用ディスク容量のパーセンテージ。                                                                                                                                                                                                                                                               |
| MaxDiskUsedPct          | 現在のBEのすべてのディスクの中で最も高い使用ディスク容量のパーセンテージ。                                                                                                                                                                                                                                             |
| RemoteUsedCapacity      | 現在のBEがホット・コールドティアリング機能を使用した後にリモートストレージにアップロードされたデータが占有するディスク容量。                                                                                                                                                                        |
| Tag                     | 現在のBEのタグ情報、JSON形式で表示される。保存されるタグ情報は、デプロイメントモードによって異なる。統合ストレージ・コンピューティングモードでは、現在のBEリソースグループの名前が保存される。分離ストレージ・コンピューティングモードでは、追加の情報が保存される。 |
| ErrMsg                  | 現在のBEのハートビートが失敗した場合のエラーメッセージ。                                                                                                                                                                                                                                                                |
| Version                 | 現在のBEのバージョン情報。                                                                                                                                                                                                                                                                                   |
| Status                  | 現在のBEのステータス情報、JSON形式で表示される。lastSuccessReportTabletsTime、lastStreamLoadTime、isQueryDisabled、isLoadDisabledなどが含まれる。異なるバージョンで保存される情報は若干異なる場合があることに注意。                                                               |
| HeartbeatFailureCounter | 現在のBEの連続したハートビート失敗回数。この回数がFE Masterで設定された`max_backend_heartbeat_failure_tolerance_count`（デフォルト値は1）を超えた場合、`Alive`フィールドはfalseに設定される。                                                                                                                                                                                             |
| NodeRole                | 現在のBEの役割。`mix`はデフォルトの役割、`computation`は現在のノードが連合分析クエリのみに使用されることを意味する。                                                                                                                                                                                                                                                                       |

## アクセス制御要件

このSQLを実行するユーザーは、少なくとも以下の権限を持つ必要があります：

| 権限  | オブジェクト | 備考 |
|------------|----|----|
| ADMIN_PRIV |    |    |

## 使用上の注意

クエリ結果をさらにフィルタリングする必要がある場合は、Table値関数[backends()](../../../sql-functions/table-valued-functions/backends.md)を使用できます。SHOW BACKENDSは以下の文と同等です：

```sql
SELECT * FROM BACKENDS();
```
