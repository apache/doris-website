## デスクリプション

このステートメントは、tableまたはパーティションのレプリカステータス情報を表示するために使用されます。

## Syntax

```sql
SHOW REPLICA DISTRIBUTION FROM [ <database_name>.]<table_name> [<partition_list>] 
[where_clause]
```
どこで：

```sql
partition_list
  : PARTITION (<partition_name>[ , ... ])
```
どこで:

```sql
where_clause
: WHERE <output_column_name> = <value>
```
## 必須パラメータ

**1. `<table_name>`**

> Tableの識別子（すなわち名前）で、データベース（Database）内で一意である必要があります。
>
> 識別子は文字で始まる必要があり（Unicode名前サポートが有効な場合はサポートされている言語の任意の文字）、識別子文字列全体がバッククォートで囲まれていない限り（例：My Object）、スペースや特殊文字を含むことはできません。
>
> 識別子は予約キーワードを使用することはできません。
>
> 詳細については、識別子要件と予約キーワードを参照してください。


## オプションパラメータ

**1. `<db_name>`**

> データベースの識別子（すなわち名前）で、クラスター（Cluster）内で一意である必要があります。
>
> 識別子は文字で始まる必要があり（Unicode名前サポートが有効な場合はサポートされている言語の任意の文字）、識別子文字列全体がバッククォートで囲まれていない限り（例：My Object）、スペースや特殊文字を含むことはできません。
>
> 識別子は予約キーワードを使用することはできません。
>
> 詳細については、識別子要件と予約キーワードを参照してください。

**2. `<partition_list>`**

> パーティション識別子（すなわち名前）のカンマ区切りリストで、Table（Table）内で一意である必要があります。
>
> 識別子は文字で始まる必要があり（Unicode名前サポートが有効な場合はサポートされている言語の任意の文字）、識別子文字列全体がバッククォートで囲まれていない限り（例：My Object）、スペースや特殊文字を含むことはできません。
>
> 識別子は予約キーワードを使用することはできません。
>
> 詳細については、識別子要件と予約キーワードを参照してください。


## 戻り値

| Column      | DataType | Note                                                                              |
|-------------|----------|-----------------------------------------------------------------------------------|
| BackendId   | Int      | レプリカが配置されているBE（Backend）ノードのIDです。                                        |
| ReplicaNum  | Int      | このBEノード上のレプリカ数です。                                                           |
| ReplicaSize | Int      | このBEノード上のすべてのレプリカが占有する総ストレージサイズ（バイト単位）です。                      |
| NumGraph    | String   | レプリカ数の割合の視覚的表現（>記号を使用）です。                                              |
| NumPercent  | String   | レプリカ数のパーセンテージ表現です（例：100.00%）。                                           |
| SizeGraph   | String   | レプリカストレージサイズの割合の視覚的表現（>記号を使用）です。                                   |
| SizePercent | String   | レプリカストレージサイズのパーセンテージ表現です（例：100.00%）。                                |

## アクセス制御要件

このSQLコマンドを実行するユーザーは、少なくとも以下の権限を持っている必要があります：

| Privilege  | Object   | 注釈                                                                                                                            |
|:-----------|:---------|:---------------------------------------------------------------------------------------------------------------------------------|
| Admin_priv | Database | Table、パーティション、およびシステムレベルコマンドの管理を含む、データベースでの管理操作を実行するために必要です。                           |


## 例

- 查看表的副本分布

  ```sql
  SHOW REPLICA DISTRIBUTION FROM sell_user;
  ```
  ```text
  *************************** 1. row ***************************
  BackendId: 10009
   ReplicaNum: 4
  ReplicaSize: 8857
     NumGraph: >>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>
   NumPercent: 100.00%
    SizeGraph: >>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>
  SizePercent: 100.00%
  ```
- Tableのパーティションのレプリカ分散を確認する

  ```sql
  SHOW REPLICA DISTRIBUTION FROM db1.tbl1 PARTITION(p1, p2);
  ```
