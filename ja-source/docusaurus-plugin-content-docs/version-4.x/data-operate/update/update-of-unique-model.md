このドキュメントでは、さまざまなロード方法を使用してDorisユニークキーモデルでデータを更新する方法について説明します。

## 全行更新

Stream Load、Broker Load、Routine Load、Insert Intoなど、Dorisでサポートされている方法を使用してユニークキーモデルにデータをロードする場合、既存のプライマリキーのデータ行が存在しない場合は新しいデータが挿入されます。既存のプライマリキーのデータ行が存在する場合は、そのデータが更新されます。これは、Dorisユニークキーモデルでのロード操作が「upsert」モードで動作することを意味します。既存レコードの更新プロセスは、デフォルトで新しいレコードをロードする場合と同じであるため、詳細についてはデータロードのドキュメントを参照してください。

## 部分列更新

Unique Key ModelTableの部分列更新に関する詳細情報（使用例、柔軟な部分列更新、新しい行の処理を含む）については、[Partial Column アップデート](./partial-column-update.md#partial-column-update-for-unique-key-model)を参照してください。
