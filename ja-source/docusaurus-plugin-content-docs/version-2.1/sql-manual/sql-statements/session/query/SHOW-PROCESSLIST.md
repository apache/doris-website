## 説明

ユーザーが実行しているスレッドを表示します。  
このコマンドは現在のFEの接続情報のみを表示できます。  
クラスター全体の接続情報を表示する必要がある場合は、セッション変数を追加する必要があります：

```
SET SHOW_ALL_FE_CONNECTION = TRUE;
```
## 構文

```sql
SHOW [FULL] PROCESSLIST
```
## オプションパラメータ

**1. `FULL`**

> 他のユーザーの接続情報を表示するかどうかを示す

## 戻り値

| カラム           | 説明                                   |
|------------------|--------------------------------------|
| CurrentConnected | 接続が現在のものかどうか                              |
| Id               | このスレッドの一意識別子                            |
| User             | このスレッドを開始したユーザー                            |
| Host             | リクエストを送信するクライアントのIPアドレスとポート番号が記録される                |
| LoginTime        | 接続を確立した時刻                              |
| Catalog          | 現在コマンドが実行されているデータディレクトリ                    |
| Db               | コマンドが実行されているデータベース。データベースが指定されていない場合、値はNULL |
| Command          | この瞬間にスレッドが実行しているコマンド                         |
| Time             | 前のコマンドが現在の状態に送信された時刻（秒単位）                 |
| State            | スレッドの状態                                |
| QueryId          | 現在のクエリ文のID                           |
| Info             | 一般的に、スレッドによって実行された文が記録され、デフォルトでは最初の100文字のみが表示される         |

一般的なCommandタイプは以下の通りです：

| カラム | 説明 |
| -- | -- |
| Query | スレッドが文を実行中 |
| Sleep | クライアントが実行文を送信するのを待機中 |
| Quit | スレッドが終了中 |
| Kill | kill文を実行中 |

## アクセス制御要件

このSQLコマンドを実行するユーザーは、少なくとも以下の権限を持つ必要があります：

| 権限    | オブジェクト    | 備考                      |
|:-------------|:----------|:---------------------------|
| ADMIN_PRIV        | DATABASE  | 他のユーザーの接続情報を表示したい場合、ADMIN権限が必要 |
