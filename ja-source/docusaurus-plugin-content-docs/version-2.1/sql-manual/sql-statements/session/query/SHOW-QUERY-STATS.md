## 説明

このステートメントは、履歴クエリによってヒットしたデータベーステーブルの列を表示するために使用されます

## 構文

```sql
SHOW QUERY STATS [ { [FOR <db_name>] | [FROM <table_name>] } ] [ALL] [VERBOSE]];
```
## オプションパラメータ

**1. `<db_name>`**

> このパラメータが設定されている場合、データベースのマッチング情報が表示されます

**2. `<table_name>`**

> このパラメータが設定されている場合、テーブルのマッチング状況が照会されます

**3. `ALL`**

> ALLはすべてのインデックスのマッチング情報を表示するかどうかを指定します

**4. `VERBOSE`**

> VERBOSEは詳細なマッチング情報を表示します

## アクセス制御要件

このSQLコマンドを実行するユーザーは、少なくとも以下の権限を持っている必要があります：

| 権限                | オブジェクト | 備考                              |
|:-------------------|:----------|:----------------------------------|
| SELECT_PRIV        | DATABASE  | 照会されるデータベースに対してSELECT権限を持っている必要があります |

## 使用上の注意

- データベースとテーブルの履歴クエリマッチング状況を照会します。データはfe再起動後にリセットされ、統計は各feごとに収集されます。

- FOR DATABASEとFROM TABLEを使用して、照会するデータベースまたはテーブルのマッチング情報を指定でき、その後にデータベース名またはテーブル名が続きます。

- ALLとVERBOSEパラメータは単独でも組み合わせでも使用できますが、最後に使用する必要があり、テーブルクエリにのみ適用されます。

- 使用中のデータベースがない場合、単純に'SHOW QUERY STATS'を実行すると、すべてのデータベースのヒット数が表示されます。

- ヒット結果には2つの列がある場合があります：QueryCountは列が照会された回数を示し、FilterCountは列がwhere条件として照会された回数を示します。
