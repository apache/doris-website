## 説明

このステートメントは、テーブルまたはパーティションのレプリカステータス情報を表示するために使用されます。

## 構文

```sql
SHOW REPLICA DISTRIBUTION FROM [ <database_name>.]<table_name> [<partition_list>] 
[where_clause]
```
ここで：

```sql
partition_list
  : PARTITION (<partition_name>[ , ... ])
```
ここで：

```sql
where_clause
: WHERE <output_column_name> = <value>
```
## 必須パラメータ

**1. `<table_name>`**

> テーブルの識別子（名前）で、Database内で一意である必要があります。
>
> 識別子は文字から始まる必要があり（Unicode名前サポートが有効な場合はサポート言語の任意の文字）、識別子文字列全体がバッククォートで囲まれていない限り（例：My Object）、スペースや特殊文字を含むことはできません。
>
> 識別子は予約キーワードを使用できません。
>
> 詳細については、識別子要件と予約キーワードを参照してください。


## オプションパラメータ

**1. `<db_name>`**

> データベースの識別子（名前）で、Cluster内で一意である必要があります。
>
> 識別子は文字から始まる必要があり（Unicode名前サポートが有効な場合はサポート言語の任意の文字）、識別子文字列全体がバッククォートで囲まれていない限り（例：My Object）、スペースや特殊文字を含むことはできません。
>
> 識別子は予約キーワードを使用できません。
>
> 詳細については、識別子要件と予約キーワードを参照してください。

**2. `<partition_list>`**

> パーティション識別子（名前）のカンマ区切りリストで、Table内で一意である必要があります。
>
> 識別子は文字から始まる必要があり（Unicode名前サポートが有効な場合はサポート言語の任意の文字）、識別子文字列全体がバッククォートで囲まれていない限り（例：My Object）、スペースや特殊文字を含むことはできません。
>
> 識別子は予約キーワードを使用できません。
>
> 詳細については、識別子要件と予約キーワードを参照してください。


## 戻り値

| Column      | DataType | Note                                                                              |
|-------------|----------|-----------------------------------------------------------------------------------|
| BackendId   | Int      | レプリカが配置されているBE（Backend）ノードのID。                     |
| ReplicaNum  | Int      | このBEノード上のレプリカ数。                                           |
| ReplicaSize | Int      | このBEノード上のすべてのレプリカが占有する総ストレージサイズ（バイト単位）。       |
| NumGraph    | String   | レプリカ数の割合の視覚的表現（>記号を使用）。        |
| NumPercent  | String   | レプリカ数の割合表現、例：100.00%。                |
| SizeGraph   | String   | レプリカストレージサイズの割合の視覚的表現（>記号を使用）。 |
| SizePercent | String   | レプリカストレージサイズの割合表現、例：100.00%。         |

## アクセス制御要件

このSQLコマンドを実行するユーザーは、少なくとも以下の権限を持っている必要があります：

| Privilege  | Object   | Notes                                                                                                                            |
|:-----------|:---------|:---------------------------------------------------------------------------------------------------------------------------------|
| Admin_priv | Database | テーブル、パーティション、システムレベルコマンドの管理を含む、データベース上の管理操作を実行するために必要。 |


## 例

- 查看表的副本分布

  ```sql
  SHOW REPLICA DISTRIBUTION FROM sell_user;
  ```
  ```text
  *************************** 1. row ***************************
  BackendId: 10009
   ReplicaNum: 4
  ReplicaSize: 8857
     NumGraph: >>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>
   NumPercent: 100.00%
    SizeGraph: >>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>
  SizePercent: 100.00%
  ```
- テーブルのパーティションのレプリカ分散を確認する

  ```sql
  SHOW REPLICA DISTRIBUTION FROM db1.tbl1 PARTITION(p1, p2);
  ```
