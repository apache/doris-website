## 説明

カスタムキーを削除します。削除するキーの名前は完全に一致している必要があります。

## 構文

```sql
DROP ENCRYPTKEY [IF EXISTS] <key_name>
```
## 必須パラメータ

**1. `<key_name>`**

> 削除するキーの名前を指定します。データベース識別子を含む場合があります。
> 例: `db1.my_key`

## オプションパラメータ

**1. `[IF EXISTS]`**

> 指定した場合、存在しないキーの削除を試行してもエラーが発生しません。

## アクセス制御要件

このSQLコマンドを実行するユーザーは、以下の最小権限を持つ必要があります：

| 権限         | オブジェクト    | 備考                                                                  |
|:-------------|:-------------|:---------------------------------------------------------------------|
| `ADMIN_PRIV` | User / Role  | キー削除操作を実行するには、ユーザーまたはロールが`ADMIN_PRIV`権限を持つ必要があります |

## 例

- キーを削除する

  ```sql
  DROP ENCRYPTKEY my_key;
  ```
- 存在しない場合でもエラーを発生させずにキーを削除する

  ```sql
  DROP ENCRYPTKEY IF EXISTS testdb.my_key;
  ```
