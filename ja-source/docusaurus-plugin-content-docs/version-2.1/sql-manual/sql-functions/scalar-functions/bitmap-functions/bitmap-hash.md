## 説明

任意の入力タイプの32ビットハッシュ値を計算し、そのハッシュ値を含むBitmapを返します。

## 構文

```sql
BITMAP_HASH(<expr>)
```
## パラメータ

| パラメータ | 説明           |
|-----------|-----------------------|
| `<expr>`  | 任意の値またはフィールド式 |

## 戻り値

パラメータ `<expr>` の32ビットハッシュ値を含むBitmapを返します。

::: note

使用されるハッシュアルゴリズムはMurMur3です。  
MurMur3は高性能で衝突の少ないハッシュアルゴリズムであり、ランダム分布に近い値を生成し、カイ二乗分布検定に合格できます。計算されるハッシュ値は、異なるハードウェアプラットフォームやシード値によって異なる場合があることにご注意ください。  
このアルゴリズムの性能の詳細については、[Smhasher](http://rurban.github.io/smhasher/)ベンチマークを参照してください。

:::

## 例

値のMurMur3ハッシュを計算するには、次のように使用できます：

```sql
select bitmap_to_array(bitmap_hash('hello'))[1];
```
結果は次のようになります：

```text
+-------------------------------------------------------------+
| %element_extract%(bitmap_to_array(bitmap_hash('hello')), 1) |
+-------------------------------------------------------------+
|                                                  1321743225 |
+-------------------------------------------------------------+
```
一部のシナリオにおいて`count distinct`よりも効率的である可能性があるbitmapを使用して、列内の異なる値を数えるには：

```sql
select bitmap_count(bitmap_union(bitmap_hash(`word`))) from `words`;
```
結果は次のようになります：

```text
+-------------------------------------------------+
| bitmap_count(bitmap_union(bitmap_hash(`word`))) |
+-------------------------------------------------+
|                                        33263478 |
+-------------------------------------------------+
```
