## 説明

この関数は、JSONドキュメントからフィールドの値を抽出し、INT型に変換するために使用されます。この関数は、指定されたパス上のフィールドの値を返すか、値をINT型に変換できない場合やパスが指す フィールドが存在しない場合はNULLを返します。

## 構文

```sql
GET_JSON_INT( <json_str>, <json_path>)
```
## 必須パラメータ

| parameters| described|
|------|------|
| `<json_str>`| データを抽出するJSONストリングが必要です。 |
| `<json_path>`| フィールドの場所を指定するJSONパス。パスはドット記法で表記できます。 |

## 使用上の注意

jsonストリング内の指定されたパスの整数コンテンツを解析し取得します。
`<json_path>`は$シンボルで始まり、パススプリッターとして.を使用する必要があります。パスに.が含まれる場合は、二重引用符で囲むことができます。
[ ]を使用して配列インデックスを表し、0から開始します。
pathのコンテンツには"、[、]を含めることはできません。
`<json_str>`フォーマットが正しくない場合、または`<json_path>`フォーマットが正しくない場合、またはマッチが見つからない場合はNULLを返します。
また、同じ機能を実現するためにjsonbタイプとjsonb_extract_XXX関数を使用することを推奨します。
特殊な状況は以下のように処理されます：
- JSONに指定されたフィールドが存在しない場合、`<json_path>`NULLを返します
- `<json_path>`がJSONで指定するフィールドの実際のタイプがjson_extract_tで指定されたタイプと一致しない場合、指定されたタイプtに無損失で変換できる場合は指定されたタイプtを返し、変換できない場合はNULLを返します。

## 例

1. キーが"k1"の値を取得

```sql
SELECT get_json_int('{"k1":1, "k2":"2"}', "$.k1");
```
```sql

+--------------------------------------------+
| get_json_int('{"k1":1, "k2":"2"}', '$.k1') |
+--------------------------------------------+
|                                          1 |
+--------------------------------------------+
```
2. キー"my.key"を持つ配列の2番目の要素を取得します

```sql
SELECT get_json_int('{"k1":"v1", "my.key":[1, 2, 3]}', '$."my.key"[1]');
```
```sql
+------------------------------------------------------------------+
| get_json_int('{"k1":"v1", "my.key":[1, 2, 3]}', '$."my.key"[1]') |
+------------------------------------------------------------------+
|                                                                2 |
+------------------------------------------------------------------+

3. Gets the first element in an array with secondary path k1.key -> k2

```sql
SELECT get_json_int('{"k1.key":{"k2":[1, 2]}}', '$."k1.key".k2[0]');

 ```

 ```sql
+--------------------------------------------------------------+
| get_json_int('{"k1.key":{"k2":[1, 2]}}', '$."k1.key".k2[0]') |
+--------------------------------------------------------------+
|                                                            1 |
+--------------------------------------------------------------+

```
