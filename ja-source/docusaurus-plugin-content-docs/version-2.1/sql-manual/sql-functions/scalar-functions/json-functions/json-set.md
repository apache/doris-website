## 説明
json_set関数は、JSONにデータを挿入または更新し、結果を返します。

## 構文

```sql
JSON_SET (<json_str>,  <path>,  <val> [, <path>,  <val>, ...])
```
## パラメーター

| パラメーター    | 説明                                                                                                                      |
|-------|-------------------------------------------------------------------------------------------------------------------------|
| `<jsonStr>` | 挿入されるJSONオブジェクト。NULL を含む任意の型の要素を持つJSONオブジェクトを指定できます。要素が指定されない場合は、空の配列が返されます。json_strが有効なJSONでない場合、またはいずれかのpathパラメーターが有効なpath式でないか、* ワイルドカードを含む場合は、エラーが返されます。 |
| `<jsonPath>` | 挿入されるJSONパス。NULLの場合、NULLを返します。                                                                                       |
| `<val>` | JSONに挿入される値。NULLの場合、対応する位置にNULL値が挿入されます。                                                                    |

## 戻り値
JSON値を返します。

`json_set`関数は、JSON内のデータを挿入または更新し、結果を返します。`json_str`または`path`がNULLの場合はNULLを返します。それ以外の場合、`json_str`引数が有効なJSONでないか、いずれかのpath引数が有効なpath式でないか、* ワイルドカードを含む場合にエラーが発生します。

path-value ペアは左から右へと評価されます。

json内の既存のパスに対するpath-valueペアは、既存のjson値を新しい値で上書きします。json内の存在しないパスに対するpath-valueペアは、パスが以下のいずれかのタイプの値を特定する場合にjsonに値を追加します：

* 既存のオブジェクトに存在しないメンバー。メンバーはオブジェクトに追加され、新しい値に関連付けられます。

* 既存の配列の末尾を超える位置。配列は新しい値で拡張されます。既存の値が配列でない場合、配列として自動ラップされ、その後新しい値で拡張されます。

それ以外の場合、json内の存在しないパスに対するpath-valueペアは無視され、効果はありません。

## 例

```sql
select json_set(null, null, null);
```
```text
+------------------------------+
| json_set(NULL, NULL, 'NULL') |
+------------------------------+
| NULL                         |
+------------------------------+
```
```sql
select json_set('{"k": 1}', "$.k", 2);
``` 
```text
+------------------------------------+
| json_set('{\"k\": 1}', '$.k', '2') |
+------------------------------------+
| {"k":2}                            |
+------------------------------------+
```
```sql
select json_set('{"k": 1}', "$.j", 2);
```
```text
+------------------------------------+
| json_set('{\"k\": 1}', '$.j', '2') |
+------------------------------------+
| {"k":1,"j":2}                      |
+------------------------------------+
```
