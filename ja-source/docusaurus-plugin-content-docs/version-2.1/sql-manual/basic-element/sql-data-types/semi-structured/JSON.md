# JSON

## JSON

JSONデータ型は[JSON](https://www.rfc-editor.org/rfc/rfc8785)データをバイナリ形式で効率的に保存し、JSON関数を通じてその内部フィールドへのアクセスを可能にします。

デフォルトでは、最大1048576バイト（1MB）をサポートし、最大2147483643バイト（2GB）まで増加させることができます。これは`string_type_length_soft_limit_bytes`設定で調整できます。

通常のSTRING型でJSON文字列を保存することと比較して、JSON型には2つの主な利点があります：
1. データ挿入時のJSON形式検証。
2. より効率的なバイナリ保存形式で、`get_json_xx`関数と比較して`json_extract`などの関数を使用したJSON内部フィールドへのより高速なアクセスを可能にします。

**注記**: バージョン1.2.xでは、JSON型はJSONBという名前でした。MySQLとの互換性を保つため、バージョン2.0.0から名前がJSONに変更されました。古いテーブルでは従来の名前を使用することができます。

### 構文

**定義:**

```sql
json_column_name JSON
```
**挿入:**
- `INSERT INTO VALUES`を使用し、クォートで囲まれた文字列としてフォーマットを指定します。例：

```sql
INSERT INTO table_name(id, json_column_name) VALUES (1, '{"k1": "100"}')
```
- STREAM LOADの場合、対応する列の形式は追加の引用符なしの文字列です。例：

```
12	{"k1":"v31", "k2": 300}
13	[]
14	[123, 456]
```
- JSON内で`'\r'`、`'\t'`などの`'\'`を含むすべての特殊文字が現れる場合、replace関数を使用して`"\"`を`"\\"`に置き換える必要があります。例えば、`"\n"`を`"\\n"`に置き換える必要があります


**Query:**
- JSON列全体を直接選択する：

```sql
SELECT json_column_name FROM table_name;
```
- JSON関数を使用してJSONから特定のフィールドやその他の情報を抽出します。例：

```sql
SELECT json_extract(json_column_name, '$.k1') FROM table_name;
```
- JSON型は整数、文字列、BOOLEAN、ARRAY、MAPとの間でキャストできます。例：

```sql
SELECT CAST('{"k1": "100"}' AS JSON);
SELECT CAST(json_column_name AS STRING) FROM table_name;
SELECT CAST(json_extract(json_column_name, '$.k1') AS INT) FROM table_name;
```
:::tip
JSON型は現在、`GROUP BY`、`ORDER BY`、または比較操作には使用できません。
:::

### example
テーブル作成、データロード、クエリを含むJSONデータ型のチュートリアル。

#### データベースとテーブルの作成

```
CREATE DATABASE testdb;

USE testdb;

CREATE TABLE test_json (
  id INT,
  j JSON
)
DUPLICATE KEY(id)
DISTRIBUTED BY HASH(id) BUCKETS 10
PROPERTIES("replication_num" = "1");
```

### FAQ

1. JSONのnullとSQL のNULL（つまり、IS NULL）には違いがありますか？

はい、違いがあります。JSONでは、null（例：{"key1": null}）はキーkey1が存在し、その値が明示的にnullであることを意味します。これはJSONバイナリにエンコードされる特別な型です。

対照的に、SQL NULL（IS NULLを使用する場合）は、キーがJSONオブジェクト内に全く存在しないことを示す場合があります。

例：

``` sql
mysql> SELECT JSON_EXTRACT_STRING('{"key1" : null}', "$.key1") IS NULL;
+----------------------------------------------------------+
| JSON_EXTRACT_STRING('{"key1" : null}', "$.key1") IS NULL |
+----------------------------------------------------------+
|                                                        0 |
+----------------------------------------------------------+
1 row in set (0.00 sec)

mysql> SELECT JSON_EXTRACT_STRING('{"key1" : null}', "$.key_not_exist") IS NULL;
+-------------------------------------------------------------------+
| JSON_EXTRACT_STRING('{"key1" : null}', "$.key_not_exist") IS NULL |
+-------------------------------------------------------------------+
|                                                                 1 |
+-------------------------------------------------------------------+
1 row in set (0.01 sec)
```
2. `GET_JSON_XXX`関数と`JSON_EXTRACT_XXX`関数の違いは何ですか？どちらを選択すべきでしょうか？

`GET_JSON_XXX`関数は文字列型での使用を目的として設計されており、生のJSON文字列から直接値を抽出します。一方、`JSON_EXTRACT_XXX`関数はJSONデータ型専用に実装されており、そのために最適化されています。
