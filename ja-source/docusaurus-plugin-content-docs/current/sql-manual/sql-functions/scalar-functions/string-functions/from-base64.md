## 説明

FROM_BASE64関数は、Base64エンコードされた文字列を元の文字列にデコードします。これはTO_BASE64の逆操作であり、RFC 4648標準に従います。Base64エンコーディングは、テキストプロトコルでバイナリデータを送信するために一般的に使用され、この関数はエンコードされたデータを復元できます。標準のBase64文字セット（A-Z、a-z、0-9、+、/）とパディング文字（=）をサポートします。

## 構文

```sql
FROM_BASE64(<str>)
```
## パラメータ

| パラメータ | 説明 |
|---------|-----------------|
| `<str>` | デコードするBase64エンコード文字列。型: VARCHAR |

## 戻り値

VARCHAR型を返し、Base64デコード後の元の文字列を表します。

デコードルール:
- 標準のBase64文字セットを受け入れます: A-Z, a-z, 0-9, +, /
- パディング文字 = をサポート
- RFC 4648標準に準拠
- 空白文字（スペース、改行など）を自動的に無視
- デコード結果には印刷不可能文字が含まれる場合があります

特殊ケース:
- 入力がNULLの場合、NULLを返します
- 入力に不正なBase64文字が含まれている場合、NULLを返します
- 入力が空文字列の場合、空文字列を返します
- パディングが正しくない、または形式が無効な場合、NULLを返します

## 例

1. 基本的なデコード

```sql
SELECT FROM_BASE64('MQ=='), FROM_BASE64('QQ==');
```
```text
+---------------------+---------------------+
| FROM_BASE64('MQ==') | FROM_BASE64('QQ==') |
+---------------------+---------------------+
| 1                   | A                   |
+---------------------+---------------------+
```
2. マルチ文字デコーディング

```sql
SELECT FROM_BASE64('MjM0'), FROM_BASE64('SGVsbG8=');
```
```text
+---------------------+-------------------------+
| FROM_BASE64('MjM0') | FROM_BASE64('SGVsbG8=') |
+---------------------+-------------------------+
| 234                 | Hello                   |
+---------------------+-------------------------+
```
3. NULL値の処理

```sql
SELECT FROM_BASE64(NULL);
```
```text
+-------------------+
| FROM_BASE64(NULL) |
+-------------------+
| NULL              |
+-------------------+
```
4. 空文字列の処理

```sql
SELECT FROM_BASE64('');
```
```text
+-----------------+
| FROM_BASE64('') |
+-----------------+
|                 |
+-----------------+
```
5. 不正文字の処理

```sql
SELECT FROM_BASE64('!!!'), FROM_BASE64('ABC@DEF');
```
```text
+--------------------+------------------------+
| FROM_BASE64('!!!') | FROM_BASE64('ABC@DEF') |
+--------------------+------------------------+
| NULL               | NULL                   |
+--------------------+------------------------+
```
