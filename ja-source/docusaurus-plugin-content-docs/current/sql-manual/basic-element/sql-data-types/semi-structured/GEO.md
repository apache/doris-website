地理空間型は、地理空間データの格納と操作に使用される特殊なデータベースデータ型で、点、線、多角形などの幾何オブジェクトを表現でき、主要な目的は以下の通りです：
- 地理的位置情報の格納（例：経度と緯度）
- 空間クエリのサポート（例：距離計算、エリア包含、交差判定）
- 地理空間解析の処理（例：バッファ解析、経路計画）
地理情報システムは、地図サービス、物流スケジューリング、位置ベースソーシャルネットワーキング、気象監視などで広く使用されています。中核的な要件は、大量の空間データを効率的に格納し、低レイテンシの空間計算をサポートすることです。


## コアエンコーディング技術
### S2 Geometry Library
S2 GeometryはGoogleが開発した球面幾何エンコーディングシステムです。そのコアアイデアは、球面から平面への投影を通じて、グローバルな地理空間データの効率的なインデックスを実現することです。

#### コア原理
- 球面投影：地球の球面を正六面体の6つの面に投影し、3D球面データを2D平面データに変換します。
- 階層グリッド分割：各面を再帰的に四角形グリッド（セル）に分割し、各セルをさらに4つのより小さなサブセルに細分割でき、30レベルの精度を持つ階層構造を形成します（レベルが高いほど、セルエリアが小さく、精度が高くなります）。
- 64ビットエンコーディング：各セルに一意の64ビットIDが割り当てられ、これにより空間位置を迅速に特定し、空間関係を判定できます。
- Hilbert曲線オーダリング：Hilbert空間充填曲線を使用してセルをエンコードし、空間的に隣接するセルが連続したIDを持つようにして、範囲クエリのパフォーマンスを最適化します。

#### 利点
- 高精度でスムーズな遷移：30レベルの階層で、グローバル（レベル0）からセンチメートルレベル（レベル30）までの精度範囲を提供し、スムーズな遷移を保証して異なるシナリオのニーズを満たします。
- グローバル範囲クエリの効率性：大規模な空間クエリ（例：大陸間、国家間地域解析）に適しており、パフォーマンスの大幅な劣化がありません。
- 効率的な空間関係計算：セルIDを通じて包含、交差などの関係を迅速に判定でき、複雑な幾何演算を回避できます。


### GeoHashエンコーディング
GeoHashは等角直線投影に基づく地理符号化方法で、経度と緯度を文字列に変換することで空間インデックスを実現します。

#### コア原理
- 平面投影：地球の球面を平面として近似し、経度と緯度の二進分割によってエリアを再帰的に分割します。
- 矩形グリッド分割：地球表面を異なる精度の矩形セルに分割します。文字列の長さが精度を決定し（最大12文字）、文字を1つ追加するごとに精度が約10倍向上します。
- Z-order曲線エンコーディング：経度と緯度のバイナリビットを交互に切り詰めてZ-order曲線を形成し、2D座標を1D文字列に変換します。

#### 特徴
- インデックスの便利さ：文字列プレフィックスマッチングを通じて隣接エリアを迅速にクエリできます（例：同じプレフィックスを持つGeoHashコードは空間的に隣接するエリアに対応）。
- 制限事項：
  - 制限された精度レベル：最大12レベルで、レベル間の急激な遷移により、高精度なスムーズ分割のニーズを満たすことが困難です。
  - Z-order曲線の可変性：曲線のジャンプにより、空間的に隣接するエリアが不連続なコードを持つ可能性があり、範囲クエリの精度に影響します。
  - 大規模クエリでの低効率：グローバル範囲をクエリする際、大量の離散セルをスキャンする必要があり、パフォーマンスが低下します。


### 総合比較と選択
S2 Geometry LibraryとGeoHashの特性を総合的に比較して、地理空間処理のサードパーティ依存関係としてS2 Geometry Libraryを選択します。主な理由は以下の通りです：
- グローバル範囲クエリへの適応性：S2の階層グリッド設計は大規模な空間解析により適しており、GeoHashはクロスリージョンクエリでパフォーマンスボトルネックがあります。
- 精度とスムーズさ：S2の30レベル階層は、グローバルからセンチメートルレベルまでのスムーズな遷移を実現でき、マルチシナリオ精度要件を満たし、GeoHashの12レベル分割より優れています。
- 空間連続性：Hilbert曲線はZ-order曲線よりも優れた空間連続性を持ち、範囲クエリでの冗長計算を削減できます。


## WKTの紹介
WKT（Well-Known Text）は、地理空間データを表現するための標準テキスト形式です。

### 定義
- テキスト形式：テキスト文字列で幾何オブジェクトの構造と座標を記述します。
- 特徴：人間が読みやすく、編集が容易で、手動入力や簡単なデータ交換に適しています。

### 構文構造
- 基本形式：GeometryType(CoordinateValues)
- 一般的な幾何型：
  - Point：POINT(経度, 緯度)  
    例：POINT(112.46, 45.23)は点の経度と緯度を表します。
  - LineString：LINESTRING(point1, point2)  
    例：LINESTRING(0 0, 1 1)は2つの点を結ぶ線分を表します。
  - Polygon：POLYGON ((0 0, 10 0, 10 10, 0 10, 0 0))


## WKBの紹介
WKB（Well-Known Binary）は、地理空間データを表現するための標準バイナリデータ形式です。

### 定義
- バイナリ形式：バイナリエンコーディングで幾何オブジェクトを表現し、WKTよりもコンパクトで効率的です。
- 特徴：コンピュータでの内部格納と送信に最適化されており、スペースを節約し、高速解析を可能にします。

### エンコーディング構造
WKBは以下の部分で構成されます：
- バイト順序（1バイト）：
  - 0x00：Big Endian（ネットワークバイト順序）
  - 0x01：Little Endian（Intel/AMDアーキテクチャで一般的）
- 幾何型（4バイト整数）：
  - 1：Point
  - 2：LineString
  - 3：Polygon
  - ...（その他の型）
- 座標値：
  - Point：x, y（またはx, y, z）
  - LineString：point1の座標, point2の座標
  - Polygon：point1の座標, point2の座標...

#### 例

```sql
    01 01 00 00 00 00 00 00 00 00 F0 3F 00 00 00 00 00 00 00 40
    └─┘ └─┘ └───────────────┘ └───────────────┘
    │   │         │               │
    Little Endian Point type     x=1.0           y=2.0
```
## GeoPoint Type
1. StringまたはVarcharを使用したWKT形式の保存

```sql
CREATE TABLE simple_point (  id INT,  wkt STRING) ;
INSERT INTO simple_point VALUES(1,'POINT(121.4737 31.2304)');

create table simple_point(id int, wkt VARCHAR(255);
INSERT INTO simple_point VALUES(1,'POINT(121.4737 31.2304)');
```
WKT形式のクエリ

```sql
select st_astext(st_geometryfromtext(wkt)) from simple_point;
+-------------------------------------+
| st_astext(st_geometryfromtext(wkt)) |
+-------------------------------------+
| POINT (121.4737 31.2304)            |
+-------------------------------------+
```
2. WKB形式を使用した格納

```sql
CREATE TABLE simple_point (  id INT,  wkb STRING) ;
INSERT INTO simple_point VALUES(1,'\x01010000005f07ce19515e5e4097ff907efb3a3f40');

create table simple_point(id int, wkb VARCHAR(255);
INSERT INTO simple_point VALUES(1,'\x01010000005f07ce19515e5e4097ff907efb3a3f40');

```
WKB形式のクエリ

```sql
select st_astext(st_geometryfromwkb(wkb)) from simple_point;
+------------------------------------+
| st_astext(st_geometryfromwkb(wkb)) |
+------------------------------------+
| POINT (121.4737 31.2304)           |
+------------------------------------+
```
3. 浮動小数点数を使用した座標の保存（xは緯度、yは経度）

```sql
CREATE TABLE simple_point_double (id INT,x DOUBLE,y DOUBLE) 
INSERT INTO simple_point_double VALUES(0,1,2);
```
浮動小数点形式のクエリ

```sql
select st_astext(st_point(x,y)) from simple_point_double;
+--------------------------+
| st_astext(st_point(x,y)) |
+--------------------------+
| POINT (1 2)              |
+--------------------------+
```

## GeoCircle型

保存方法（浮動小数点数を使用した中心座標と半径の保存）
円はWKB形式およびWKT形式に準拠しないため、中心座標（x, y）と半径（R）をそれぞれ保存するために3つの浮動小数点数が必要です：

```sql
create table simple_circle(id int, X double,Y double, R double)
INSERT INTO simple_circle VALUES(1,1.0,1.0,2);
```
クエリサークル

```sql
select st_astext(st_circle(X,Y,R)) from simple_circle;
+-----------------------------+
| st_astext(st_circle(X,Y,R)) |
+-----------------------------+
| CIRCLE ((1 1), 2)           |
+-----------------------------+
```
## 制約
### インデックス
DorisはGeo型を直接実装せず、WKTとWKBを使用して保存・変換するため、インデックス技術によるGEO型クエリのクエリ高速化は不可能です。

WKTからGEO出力への変換時は、13桁の精度のみ保証されます：

```sql
mysql> SELECT ST_AsText(ST_GeometryFromText("POINT (1 3.1415926535897223)"));
+----------------------------------------------------------------+
| ST_AsText(ST_GeometryFromText("POINT (1 3.1415926535897223)")) |
+----------------------------------------------------------------+
| POINT (1 3.14159265358972)                                     |
+----------------------------------------------------------------+
```
バイナリからGEO出力に変換する際は、13桁の精度のみが保証されます:

```sql
mysql> select ST_AsText(ST_GeomFromWKB(ST_AsBinary(ST_Point(24.7,3.141592653589793))));
+--------------------------------------------------------------------------+
| ST_AsText(ST_GeomFromWKB(ST_AsBinary(ST_Point(24.7,3.141592653589793)))) |
+--------------------------------------------------------------------------+
| POINT (24.7 3.1415926535898)                                             |
+--------------------------------------------------------------------------+
```
