## VARBINARY
### 説明
可変長バイナリバイト列で、Mは最大長（バイト単位）を表します。

VARCHARとは異なり、文字セットや照合順序を持たずにデータをバイト単位で格納・比較するため、任意のバイナリデータ（ファイル断片、ハッシュ、暗号化/圧縮データなど）に適しており、最大長は2,147,483,647です。

現在、VARBINARYカラムは作成されたマテリアライズドビューでサポートされておらず、Group Byキー、Join Keys、比較述語での使用もサポートされていません。

バージョンと制限：4.0以降でサポート。現在Dorisでのテーブル作成と格納のカラム型としてはサポートされていません。外部システムのBINARY/VARBINARYフィールドをCatalogs経由でDoris VARBINARYにマッピングしてクエリすることは可能です。
