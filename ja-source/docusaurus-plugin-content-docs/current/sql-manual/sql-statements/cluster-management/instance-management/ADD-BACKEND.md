## 説明

ADD BACKENDは、DorisクラスターにBEノードを1つ以上追加するために使用されます。このコマンドにより、管理者は新しいBEノードのホストとポートを指定し、オプションでそれらの動作を設定するプロパティを指定することができます。

## 構文

```sql
ALTER SYSTEM ADD BACKEND "<host>:<heartbeat_port>"[,"<host>:<heartbeat_port>" [, ...]] [PROPERTIES ("<key>"="<value>" [, ...] )]
```
## 必須パラメータ

**1. <host>**

> BEノードのホスト名またはIPアドレスを指定できます。

**2. <heartbeat_port>**

> BEノードのハートビートポートで、デフォルトは9050です。

## オプションパラメータ

**1. `PROPERTIES ("<key>"="<value>" [, ... ] )`**

> BEノードの追加プロパティを定義するために使用されるキーと値のペアのセットです。これらのプロパティは、追加されるBEの設定をカスタマイズするために使用できます。利用可能なプロパティには以下が含まれます：
> - `tag.location`：統合ストレージ・コンピューティングモードにおいてBEノードが属するResource Groupを指定するために使用されます。
> - `tag.compute_group_name`：分離ストレージ・コンピューティングモードにおいてBEノードが属するコンピュートグループを指定するために使用されます。
> - `tag.public_endpoint`：外部アクセス用のBEノードのパブリックエンドポイントを指定するために使用されます（例：`11.10.20.12:8010`）。これは通常、外部ユーザーアクセス用のロードバランサードメイン名またはパブリックIPです。
> - `tag.private_endpoint`：プライベートネットワークアクセス用のBEノードのプライベートエンドポイントを指定するために使用されます（例：`10.10.10.9:8020`）。これは通常、VPC内部アクセスまたはクラスター内のKubernetes Service IPに使用されます。

## アクセス制御要件

このSQLを実行するユーザーは、少なくとも以下の権限を持つ必要があります：

| Privilege | Object | Notes |
|-----------|----|-------|
| NODE_PRIV |    |       |

## 使用上の注意

1. 新しいBEノードを追加する前に、ノードが正しく設定され、実行されていることを確認してください。
2. [Resource Group](../../../../admin-manual/workload-management/resource-group.md)を使用することで、クラスター内のBEノードをより良く管理・整理することができます。
3. 複数のBEノードを追加する際は、効率を向上させるために1つのコマンドで指定できます。
3. BEノードを追加した後、[`SHOW BACKENDS`](./SHOW-BACKENDS.md)を使用して、正常に追加され、正常な状態にあるかどうかを確認してください。
4. クラスターの可用性と耐障害性を向上させるため、異なる物理的な場所やラックにBEノードを追加することを検討してください。
5. 新しく追加されたBEノードが適切に利用されるよう、定期的にクラスター内の負荷をチェック・バランシングしてください。

## 例

1. 追加プロパティなしでBEノードを追加

   ```sql
   ALTER SYSTEM ADD BACKEND "192.168.0.1:9050,192.168.0.2:9050";
   ```
このコマンドは、クラスターに2つのBEノードを追加します：
   * 192.168.0.1、ポート9050
   * 192.168.0.2、ポート9050
   追加のプロパティは指定されていないため、デフォルト設定が適用されます。

2. ストレージ・コンピューティング統合モードにおいて、指定されたResource GroupにBEノードを追加する

   ```sql
   ALTER SYSTEM ADD BACKEND "doris-be01:9050" PROPERTIES ("tag.location" = "groupb");
   ```
このコマンドは、単一のBEノード（ホスト名doris-be01、ポート9050）をクラスター内のResource Group `groupb`に追加します。

3. ストレージとコンピューティングの分離モードでは、指定されたコンピュートグループにBEノードを追加します

   ```sql
   ALTER SYSTEM ADD BACKEND "192.168.0.3:9050" PROPERTIES ("tag.compute_group_name" = "cloud_groupc");
   ```
このコマンドは、クラスター内のコンピュートグループ `cloud_groupc` に単一のBEノード（IP 192.168.0.3、ポート9050）を追加します。

4. 複雑なネットワーク環境用にパブリックエンドポイントとプライベートエンドポイントを設定したBEノードを追加する

   ```sql
   ALTER SYSTEM ADD BACKEND "192.168.1.1:9050" PROPERTIES (
       "tag.public_endpoint" = "11.10.20.12:8010",
       "tag.private_endpoint" = "10.10.10.9:8020"
   );
   ```
このコマンドは、複数のネットワークエンドポイントを持つBEノードを追加します：
   * `192.168.1.1:9050`：クラスター通信用の内部アドレス（be_host）
   * `11.10.20.12:8010`：ロードバランサー経由の外部ユーザーアクセス用のパブリックエンドポイント
   * `10.10.10.9:8020`：VPC内部またはKubernetesクラスター間アクセス用のプライベートエンドポイント

   この設定は、BEノードが異なるネットワークコンテキストからアクセス可能である必要があるクラウド環境やKubernetesクラスターで有用です。詳細については、[Stream Load in Complex Network Environments](../../../../data-operate/import/load-internals/stream-load-in-complex-network.md)を参照してください。
