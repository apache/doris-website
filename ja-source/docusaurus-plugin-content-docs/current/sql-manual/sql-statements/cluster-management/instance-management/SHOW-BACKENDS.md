## 説明

このステートメントは、BEノードの基本ステータス情報を表示するために使用されます。

## 構文

```sql
 SHOW BACKENDS
```
## 戻り値

| Column                      | Note                                                                                                                                                                                                                                                                                                                         |
|-------------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| BackendId               | 現在のBEのID。                                                                                                                                                                                                                                                                                                                    |
| Host                    | 現在のBEのIPアドレスまたはホスト名。                                                                                                                                                                                               |
| HeartbeatPort           | 現在のBEのハートビートサービスの通信ポート。                                                                                                                                                                           |
| BePort                  | 現在のBEのthrift RPC通信ポート。                                                                                                                                                                         |
| HttpPort                | 現在のBEのHTTP通信ポート。                                                                                                                                                                               |
| BrpcPort                | 現在のBEのbRPC通信ポート。                                                                                                                                                                               |
| ArrowFlightSqlPort      | 現在のBEのArrowFlightプロトコルの通信ポート。                                                                                                                                                                        |
| LastStartTime           | 現在のBEが開始されたタイムスタンプ。                                                                                                                                                                                   |
| LastHeartbeat           | 現在のBEから送信された最後の成功したハートビートのタイムスタンプ。                                                                                                                                                                       |
| Alive                   | 現在のBEが生きているかどうか。                                                                                                                                                                             |
| SystemDecommissioned    | この値がtrueの場合、現在のBEノードが安全な廃止処理中であることを意味します。                                                                                                                                                                                        |
| TabletNum               | 現在のBEに保存されているタブレットの数。                                                                                                                                                                              |
| DataUsedCapacity        | 現在のBEのデータが占有するディスク容量。                                                                                                                                                                       |
| TrashUsedCapacity       | 現在のBEのトラッシュ内のデータが占有するディスク容量。                                                                                                                                                                          |
| AvailCapacity           | 現在のBEの利用可能なディスク容量。                                                                                                                                                                                  |
| TotalCapacity           | 現在のBEの総ディスク容量。TotalCapacity = AvailCapacity + TrashUsedCapacity + DataUsedCapacity + その他の非ユーザーデータファイルが占有するディスク容量。                                                                                                                                                             |
| UsedPct                 | 現在のBEの総使用ディスク容量の割合。                                                                                                                                                                               |
| MaxDiskUsedPct          | 現在のBEのすべてのディスクの中で最大の使用ディスク容量の割合。                                                                                                                                                             |
| RemoteUsedCapacity      | ホット・コールド階層化機能使用後に現在のBEによってリモートストレージにアップロードされたデータが占有するディスク容量。                                                                                                        |
| Tag                     | 現在のBEのタグ情報、JSON形式で表示されます。保存されるタグ情報は異なるデプロイメントモードで異なります。統合ストレージ・コンピューティングモードでは、現在のBEリソースグループの名前が保存されます。分離ストレージ・コンピューティングモードでは、追加情報が保存されます。 |
| ErrMsg                  | 現在のBEのハートビートが失敗した際のエラーメッセージ。                                                                                                                                                                                |
| Version                 | 現在のBEのバージョン情報。                                                                                                                                                                                   |
| Status                  | 現在のBEの一部のステータス情報、JSON形式で表示され、lastSuccessReportTabletsTime、lastStreamLoadTime、isQueryDisabled、isLoadDisabledなどを含みます。異なるバージョンで保存される情報は若干異なる場合があることに注意してください。                                                               |
| HeartbeatFailureCounter | 現在のBEの連続したハートビート失敗回数。この回数がFE Masterで設定された`max_backend_heartbeat_failure_tolerance_count`（デフォルト値は1）を超えると、`Alive`フィールドがfalseに設定されます。                                                                                                                                                                                             |
| NodeRole                | 現在のBEのロール。2つのタイプがあります：`mix`はデフォルトのロールで、`computation`は現在のノードが連合分析クエリにのみ使用されることを意味します。                                                                                                                                                                                                                                                                       |

## アクセス制御要件

このSQLを実行するユーザーは、少なくとも以下の権限を持つ必要があります：

| Privilege  | Object | Notes |
|------------|----|----|
| SELECT_PRIV | internal.information_schema | すべてのユーザーはデフォルトでこのデータベースに対する権限を持っています |

## 使用上の注意

クエリ結果のさらなるフィルタリングが必要な場合は、テーブル値関数[backends()](../../../sql-functions/table-valued-functions/backends.md)を使用できます。SHOW BACKENDSは以下のステートメントと同等です：

```sql
SELECT * FROM BACKENDS();
```
