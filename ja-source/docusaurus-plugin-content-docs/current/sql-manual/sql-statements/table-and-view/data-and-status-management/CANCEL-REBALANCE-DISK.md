## 説明

`CANCEL REBALANCE DISK`文は、Backend（BE）ノードの高優先度ディスクデータバランシングをキャンセルするために使用されます。この文には以下の機能があります：

- 指定されたBEノードの高優先度ディスクバランシングをキャンセルできます。
- クラスター全体のすべてのBEノードの高優先度ディスクバランシングをキャンセルできます。
- キャンセル後も、システムはデフォルトのスケジューリング方法を使用してBEノードのディスクデータをバランシングします。

## 構文

```sql
ADMIN CANCEL REBALANCE DISK [ ON ( "<host>:<port>" [, ... ] ) ];
```
## オプションパラメータ

**1. `"<host>:<port>"`**

> 高優先度ディスクバランシングをキャンセルする必要があるBEノードのリストを指定します。
>
> 各ノードはホスト名（またはIPアドレス）とハートビートポートで構成されます。
>
> このパラメータが指定されていない場合、すべてのBEノードの高優先度ディスクバランシングがキャンセルされます。

## アクセス制御要件

このSQLコマンドを実行するユーザーは、少なくとも以下の権限を持つ必要があります：

| 権限            | オブジェクト  | 備考                                          |
| :-------------- | :---------- | :-------------------------------------------- |
| ADMIN           | System      | このコマンドを実行するには、ユーザーはADMIN権限を持つ必要があります。 |

## 使用上の注意

- このステートメントは、システムが指定されたBEのディスクデータのバランシングを優先しなくなることのみを示します。ただし、システムはデフォルトのスケジューリング方法を使用してBEのディスクデータのバランシングを継続します。
- このコマンドを実行すると、以前に設定された高優先度バランシング戦略は即座に無効になります。

## 例

- クラスター内のすべてのBEの高優先度ディスクバランシングをキャンセル：

    ```sql
    ADMIN CANCEL REBALANCE DISK;
    ```
- 指定されたBEに対する高優先度ディスクバランシングをキャンセル:

```sql
ADMIN CANCEL REBALANCE DISK ON ("192.168.1.1:1234", "192.168.1.2:1234");
```
