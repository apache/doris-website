## 説明

インポートタスクが失敗し、エラーメッセージが `ETL_QUALITY_UNSATISFIED` の場合、インポート品質に問題があることを示します。このステートメントは、品質に問題があるこれらのインポートタスクを表示するために使用されます。

## 構文

```sql
SHOW LOAD WARNINGS
[FROM <db_name>]
[
   WHERE
   [LABEL  = [ "<your_label>" ]]
   [LOAD_JOB_ID = ["<job_id>"]]
]
```
## オプションパラメータ

**1. `<db_name>`**

> `db_name`が指定されない場合、現在のデフォルトデータベースが使用されます。

**2. `<your_label>`**

> `LABEL = <your_label>`が使用される場合、指定されたラベルと正確に一致します。

**3. `<job_id>`**

> `LOAD_JOB_ID = <job_id>`が指定される場合、指定されたJOB_IDと正確に一致します。

## アクセス制御要件

このSQLコマンドを実行するユーザーは、少なくとも以下の権限を持つ必要があります：

| 権限 | オブジェクト | 備考 |
| :---------------- | :------------- | :---------------------------- |
| LOAD_PRIV | Database | データベーステーブルのインポート権限が必要です。 |

## 戻り値

指定されたデータベースのインポートタスクで品質に問題のあるデータを返します。

## 例

- 指定されたデータベースのインポートタスクで品質に問題のあるデータを表示します。ラベルは"load_demo_20210112"として指定されています。

```sql
SHOW LOAD WARNINGS FROM demo WHERE LABEL = "load_demo_20210112" 
```
