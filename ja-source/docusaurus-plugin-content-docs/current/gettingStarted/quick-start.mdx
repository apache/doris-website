# クイックスタート

:::caution 警告:

以下の迅速なデプロイメント方法は、ローカル開発およびテスト専用であり、本番環境では使用しないでください。理由は以下の通りです：

1. **データの脆弱性**: Dockerデプロイメントを使用するとデータが簡単に失われる可能性があります。コンテナが破棄されるとデータが失われるためです。単一レプリカインスタンスの手動デプロイメントにはデータの冗長性とバックアップ機能がないため、マシン障害によりデータ損失が発生する可能性があります。

2. **単一レプリカ構成**: 例のテーブル作成文はすべて単一レプリカです。本番環境では、データの信頼性を確保するためにマルチレプリカストレージを使用する必要があります。

:::

## Dockerを使用したクイックデプロイメント

Dorisバージョン2.1.8以降、Dockerを迅速なデプロイメントに使用できます。

### ステップ1: Quick-Startスクリプトのダウンロード

<a target="_blank" download rel="noopener noreferrer" href="/files/start-doris.sh"> スクリプトをダウンロード </a>し、以下のコマンドを実行して対応する実行権限を付与します。

```shell
chmod 755 start-doris.sh
```
### ステップ2: クラスターを開始する

スクリプトを実行してクラスターを開始します。デフォルトで`4.0.1`バージョンを使用します

```shell
bash start-doris.sh
```
-v パラメータを通じて起動バージョンを指定できます。例えば：

```shell
bash start-doris.sh -v 2.1.8
```
### ステップ3: MySQLクライアントを使用してクラスターに接続し、クラスターのステータスを確認する

```sql
## Check the FE status to ensure that both the Join and Alive columns are true.
mysql -uroot -P9030 -h127.0.0.1 -e 'SELECT `host`, `join`, `alive` FROM frontends()'
+-----------+------+-------+
| host      | join | alive |
+-----------+------+-------+
| 127.0.0.1 | true | true  |
+-----------+------+-------+

## Check the BE status to ensure that the Alive column is true.
mysql -uroot -P9030 -h127.0.0.1 -e 'SELECT `host`, `alive` FROM backends()'
+-----------+-------+
| host      | alive |
+-----------+-------+
| 127.0.0.1 |     1 |
+-----------+-------+

```
## ローカルクイックデプロイ

:::info 環境推奨事項：

* **オペレーティングシステム**: Ubuntu以上などのAMD/ARM主流Linux環境の使用を推奨します。
  
* **Java環境**: Java 17ランタイム環境の使用を推奨します。

* **ユーザー権限**: Linuxで新しいDorisユーザーを作成し、rootユーザーでの操作は避けることを推奨します。
  
:::

### ステップ1：バイナリパッケージのダウンロード

Apache Dorisウェブサイト[こちら](https://doris.apache.org/download)から対応するバイナリインストールパッケージをダウンロードし、展開してください。

### ステップ2：環境変数の修正

1. **システムの最大オープンファイルディスクリプタ制限の修正**

   以下のコマンドを使用して最大ファイルディスクリプタ制限を調整してください。この変更を行った後、設定を適用するためにセッションを再起動する必要があります：

   ```sql
   vi /etc/security/limits.conf 
   * soft nofile 1000000
   * hard nofile 1000000
   ```
2. **仮想メモリエリアの変更**

   以下のコマンドを使用して、仮想メモリエリアを最低2000000に恒久的に変更し、変更を即座に適用します：

   ```bash
   cat >> /etc/sysctl.conf << EOF
   vm.max_map_count = 2000000
   EOF

   ## Take effect immediately
   sysctl -p
   ```
### ステップ 3: FE をインストール

1. **FE を設定**

   FE 設定ファイル `apache-doris/fe/conf/fe.conf` の以下の内容を変更します：

   ```sql
   ## Specify Java environment
   JAVA_HOME=/home/doris/jdk

   ## Specify the CIDR block for FE listening IP
   priority_networks=127.0.0.1/32
   ```
2. **FEの開始**

   `start_fe.sh`スクリプトを実行してFEプロセスを起動します：

   ```sql
   apache-doris/fe/bin/start_fe.sh --daemon
   ```
3. **FEステータスの確認**

   MySQLクライアントを使用してクラスターに接続し、クラスターステータスを確認します：


### ステップ4: BEのインストール

1. **BEの設定**

   BE設定ファイル`apache-doris/be/conf/be.conf`で以下の内容を変更してください：

   ```sql
   ## Specify Java environment
   JAVA_HOME=/home/doris/jdk

   ## Specify the CIDR block for BE's listening IP
   priority_networks=127.0.0.1/32
   ```
2. **BEの開始**

   以下のコマンドでBEプロセスを開始します：

   ```sql
   apache-doris/be/bin/start_be.sh --daemon
   ```
3. **クラスター内でBEノードを登録**

   MySQLクライアントを使用してクラスターに接続:

   ```sql
   mysql -uroot -P9030 -h127.0.0.1
   ```
ADD BACKEND コマンドを使用して BE ノードを登録します：

   ```sql
   ALTER SYSTEM ADD BACKEND "127.0.0.1:9050";
   ```
4. **BE ステータスの確認**

## クエリの実行

1. **MySQL clientを使用してクラスターに接続:**

   ```sql
   mysql -uroot -P9030 -h127.0.0.1
   ```
2. **データベースとテストテーブルの作成:**

   ```sql
   create database demo;

   use demo; 
   create table mytable
   (
       k1 TINYINT,
       k2 DECIMAL(10, 2) DEFAULT "10.05",    
       k3 CHAR(10) COMMENT "string column",    
       k4 INT NOT NULL DEFAULT "1" COMMENT "int column"
   ) 
   COMMENT "my first table"
   DISTRIBUTED BY HASH(k1) BUCKETS 1
   PROPERTIES (
    "replication_num" = "1"
   );
   ```
3. **テストデータのインポート:**

   Insert Into文を使用してテストデータを挿入する

   ```sql
   insert into mytable values
   (1,0.14,'a1',20),
   (2,1.04,'b2',21),
   (3,3.14,'c3',22),
   (4,4.35,'d4',23);
   ```
4. **インポートされたデータを表示するために、MySQLクライアントで以下のSQLクエリを実行してください:**

   ```sql
   MySQL [demo]> select * from demo.mytable;
   +------+------+------+------+
   | k1   | k2   | k3   | k4   |
   +------+------+------+------+
   |    1 | 0.14 | a1   |   20 |
   |    2 | 1.04 | b2   |   21 |
   |    3 | 3.14 | c3   |   22 |
   |    4 | 4.35 | d4   |   23 |
   +------+------+------+------+
   4 rows in set (0.10 sec)
   ```
## FAQ

**Q: MacにDockerをインストールするにはどうすればよいですか？**

A: [Docker Desktop](https://www.docker.com/products/docker-desktop/)をダウンロードしてインストールしてください。

**Q: Mac: Docker Desktopをインストール後に「Error: Docker environment not detected」が発生する**

A: symlinkを作成してください：

```shell
sudo ln -s /Applications/Docker.app/Contents/Resources/bin/docker /usr/local/bin/docker
```
**Q: Mac: "error getting credentials - err: exit status 1, out: \`\`"**

A: このエラーは通常、Dockerクレデンシャルヘルパーの設定ミスが原因です。ローカル開発/テスト用として、回避策として`~/.docker/config.json`の`credsStore`フィールドを削除できます。注意: この回避策はクレデンシャルを平文で保存するため、ローカル開発環境でのみ推奨されます。
